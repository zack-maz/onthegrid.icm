### Broken Flows (wiring-level gaps)

1. **`stress:high|medium|low` keyword not implemented** (WAT-09 contract break)
   - Location: `src/lib/queryEvaluator.ts:351-358`
   - Cause: Only handles numeric `stress:1..10`; `parseInt('high')` returns NaN and falls through
   - Fix: Add string branch before numeric parse, mapping high→1-3, medium→4-6, low→7-10 (mirrors `severity:` handling at lines 312-314)
   - Severity: **major** — falsely satisfies a marked-complete requirement

2. **Water feed missing from StatusPanel HUD**
   - Location: `src/components/ui/StatusPanel.tsx:43-102`
   - Cause: FeedLine only rendered for flights/ships/events/sites; `waterStore.connectionStatus` is populated but never surfaced
   - Fix: Add FeedLine for water gated on `useLayerStore.activeLayers.has('water')`
   - Severity: minor — operational visibility gap, not a product break

3. **Water missing from `/health` endpoint**
   - Location: `server/routes/health.ts:7-15` `SOURCE_KEYS`
   - Cause: `water:facilities` key never added to the rollup
   - Fix: Add `water:facilities` key; same fix applies to `cron-health.ts`
   - Severity: minor — monitoring gap

4. **`/health` and `/api/cron/health` use unsafe `cacheGet`**
   - Location: `server/routes/health.ts:44`, `server/routes/cron-health.ts:42`
   - Cause: Direct `cacheGet` bypasses the 2000ms `REDIS_OP_TIMEOUT_MS` Promise.race wrapper added in PRES-23/PRES-24
   - Impact: A hung Upstash REST call can block the health probe for the full Vercel function budget, defeating the PRES-23 resilience contract
   - Fix: Wrap in `cacheGetSafe` or inline Promise.race with 2000ms timeout
   - Severity: minor — resilience gap, not a functional break

5. **AppError class defined but never thrown by routes**
   - Location: `errorHandler.ts:7-17` defines AppError; every route does `throw err` raw
   - Impact: Envelope shape is uniform (central handler wraps), but all route errors become `INTERNAL_ERROR/500` — no 502 for upstream fail, no 503 for degraded, no 400 for validation fail
   - Fix: Replace raw `throw err` in adapter error handlers (events.ts:173, sites.ts:48, news.ts:92, water.ts:80, weather.ts:41, markets.ts:71) with `throw new AppError(code, statusCode, message)`
   - Severity: minor — CLN-04 is half-realized (envelope shape ✓, code discrimination ✗)
