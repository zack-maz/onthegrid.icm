  <behavior>
    - GDELT normalizer: normalizeGdeltEvent output includes numMentions parsed from column 31, numSources parsed from column 32 (both optional, default undefined)
    - Severity scoring: computeSeverityScore(event) returns higher scores for airstrikes (weight 10) than blockades (weight 2); returns higher scores for events with more mentions; returns lower scores for older events (recency decay with ~24h half-life)
    - News matching: matchNewsToEvent(event, clusters) returns 0-3 MatchedHeadline objects; returns 0 when no clusters are within 24h temporal window; returns headlines sorted by relevance; geographic proximity (within 100km) boosts relevance score; keyword overlap between event.data.locationName and article title boosts relevance
    - Time grouping: getTimeGroup(timestamp) returns 'last_hour' for timestamps < 1h ago, 'last_day' for < 24h, 'last_week' for older
  </behavior>
  <action>
    1. **Extend ConflictEventEntity.data** in `server/types.ts`: Add `numMentions?: number` and `numSources?: number` as optional fields. These are optional so cached/backfilled data without them still validates.

    2. **Add NumSources to COL** in `server/adapters/gdelt.ts`: Add `NumSources: 32` to the COL object. In `normalizeGdeltEvent`, add `numMentions: parseInt(cols[COL.NumMentions], 10) || undefined` and `numSources: parseInt(cols[COL.NumSources], 10) || undefined` to the data object. Use `|| undefined` (not `|| 0`) so the field stays undefined when the value is 0 or NaN rather than providing a misleading 0.

    3. **Create `src/lib/severity.ts`**: Pure function `computeSeverityScore(event: ConflictEventEntity): number`. Formula: `typeWeight * Math.log2(1 + mentions) * Math.log2(1 + sources) * recencyDecay`. Type weights: airstrike=10, wmd=10, mass_violence=9, assassination=8, bombing=7, shelling=7, ground_combat=6, abduction=5, ceasefire_violation=4, assault=3, blockade=2. RecencyDecay = `1 / (1 + ageHours / 24)`. Access numMentions/numSources from event.data with fallback to 1.

    4. **Create `src/lib/newsMatching.ts`**: Pure function `matchNewsToEvent(event: ConflictEventEntity, clusters: NewsCluster[]): MatchedHeadline[]`. Export `MatchedHeadline` interface: `{ source: string; title: string; url: string }`. Algorithm: For each cluster, compute relevance = temporal score (must be within 24h, else 0) + geographic score (within 100km using haversineKm, weighted 2x) + keyword score (event.data.locationName words overlap with article title, 0.5 per match). Sort by relevance descending, take top 3. Import `haversineKm` from `@/lib/geo`.

    5. **Create `src/lib/timeGroup.ts`**: Export `TimeGroup` type (`'last_hour' | 'last_day' | 'last_week'`), `getTimeGroup(timestamp: number): TimeGroup`, `TIME_GROUP_LABELS` record, `TIME_GROUP_ORDER` array.

    6. **Write tests**:
       - Extend `server/__tests__/gdelt.test.ts` with a test that `normalizeGdeltEvent` includes numMentions and numSources when the columns contain valid numbers.
       - Create `src/__tests__/severity.test.ts`: test that airstrike scores > blockade scores; test that more mentions = higher score; test that older events score lower; test that missing numMentions defaults to 1.
       - Create `src/__tests__/newsMatching.test.ts`: test that events with no temporally-close clusters return []; test that matching returns max 3 headlines; test that geographic proximity boosts ranking.
  </action>
