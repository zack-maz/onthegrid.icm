    - Returns empty array when no sites are loaded
    - Filters to unidentified flights ONLY (data.unidentified === true)

    - Only includes alerts where haversine distance <= 50km

  <action>
    1. **Create `src/hooks/useProximityAlerts.ts`**:

    Export `ProximityAlert` interface:
    ```typescript
    export interface ProximityAlert {
      siteId: string;
      siteLat: number;
      siteLng: number;
      siteLabel: string;
      siteType: string;
      flightId: string;
      flightLabel: string;
      distanceKm: number;
      heading: number | null;
    }
    ```

    Export `useProximityAlerts(): ProximityAlert[]` hook:
    - Read `flights` from flightStore, `sites` from siteStore
    - useMemo dependent on [flights, sites]
    - Filter flights to `f.data.unidentified === true`
    - If no unidentified flights or no sites, return []
    - For each site, for each unidentified flight:
      - Coarse bbox filter: `Math.abs(flight.lat - site.lat) > 0.5` or lng same -> skip
      - Compute `haversineKm(site.lat, site.lng, flight.lat, flight.lng)`
      - If distance <= 50: add to alerts
    - Deduplicate by siteId: for each site, keep the alert with the closest flight
    - Return sorted by distanceKm ascending (most urgent first)

    Also export the pure computation function separately for testing:
    ```typescript
    export function computeProximityAlerts(
      flights: FlightEntity[],
      sites: SiteEntity[],
    ): ProximityAlert[]
    ```

    - Test: returns empty when no unidentified flights
    - Test: returns empty when flights are all identified (unidentified=false)
    - Test: returns alert when unidentified flight is within 50km of a site
    - Test: does NOT return alert when flight is > 50km away
    - Test: only returns closest flight per site when multiple are in range
    - Test: heading is included from flight data
    Use `computeProximityAlerts` directly with mock data (no store needed for pure function tests).

    1. **Create `src/components/map/ProximityAlertOverlay.tsx`**: A React component that renders proximity alert indicators on the map. Uses the `useMap` hook from `@vis.gl/react-maplibre` to project lat/lng to screen coordinates and render HTML overlays.

    **Design approach (hybrid per research recommendation):** Render small warning icons as absolutely-positioned HTML elements using `map.project([lng, lat])` to get pixel coordinates. This allows easy expand/collapse with React state, unlike pure deck.gl layers.

    Component structure:
    - Call `useProximityAlerts()` to get alerts
    - Call `useMap()` to get map instance for coordinate projection. The `useMap()` hook returns `{ current: MapRef }`. Access the map instance via `mapRef.current?.getMap()`.
    - For each alert, render a positioned div at the site's screen coordinates
    - Use `useState` to track which alert (by siteId) is expanded

    **Collapsed state:** Small warning triangle icon (Unicode warning sign or SVG), `w-6 h-6`, orange/amber background (`bg-amber-500`), rounded-full, pulsing animation (`animate-pulse`), positioned at site location with slight offset (-12px, -12px to center).

    **Expanded state (on click):** Card expands from the icon showing:
    - Site name (siteLabel)
    - "PROXIMITY ALERT" label in amber
    - Flight: flightLabel
    - Distance: distanceKm rounded to 1 decimal + "km"
    - Heading: heading degrees or "N/A"
    - Small close button (x) to collapse back

    Styling: `rounded-lg border border-amber-500/50 bg-surface-overlay/95 backdrop-blur-sm shadow-lg p-2 text-[11px]`, minimum width `w-48`.

    **Re-render on map move:** Subscribe to `map.on('move')` to update projected positions. Use a `useState` counter or `useReducer` to force re-render on move events. Throttle via RAF to avoid excessive re-renders.

    **Render as child of `<Map>`** inside BaseMap to ensure `useMap()` has access to the map context.
