## Human Verification Required

### 1. Dropdown Expand Animation

**Test:** Open the sidebar Counters section; click a counter row with a non-zero value (e.g., Unidentified or Airstrikes when live data is present)
**Expected:** Dropdown slides open smoothly below the row with a 150ms ease-in-out transition; chevron rotates 90 degrees; entity rows appear with label on left and metric on right
**Why human:** jsdom does not compute CSS transitions or `max-height` animations

### 2. Scroll Range Indicator Update

**Test:** Expand a counter row that has more than 8 entities (requires 9+ unidentified flights or 9+ airstrike events in live data); scroll the dropdown list
**Expected:** The "Showing X-Y of Z" indicator at the bottom updates in real-time as you scroll (e.g., from "Showing 1-8 of 15" to "Showing 3-10 of 15")
**Why human:** Ref-based DOM text mutation via `rangeRef.current.textContent` is not observable in unit tests; requires real scroll event

### 3. Map Fly-To Integration

**Test:** Expand any counter row with entities; click one entity item in the dropdown list
**Expected:** Map animates to center on that entity's coordinates (zoom 10); the detail panel slides in from the right showing the entity's details; the dropdown list remains open (does not close)
**Why human:** Live MapLibre map context required for fly-to; `setFlyToTarget` integration with map layer cannot be tested in jsdom
