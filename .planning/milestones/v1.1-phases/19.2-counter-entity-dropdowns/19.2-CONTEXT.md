# Phase 19.2: Counter Entity Dropdowns - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Each counter row in the Counters sidebar panel becomes expandable to reveal a scrollable list of individual entities of that subtype. Clicking an entity flies the map to it and opens its detail panel. This phase modifies `CounterRow`, `useCounterData`, and `CountersSlot` — no new panels or routes.

</domain>

<decisions>
## Implementation Decisions

### Dropdown Interaction
- Click anywhere on the CounterRow to toggle the dropdown (entire row is the tap target)
- Chevron icon on the row rotates to indicate expanded/collapsed state
- Accordion behavior — expanding one row auto-collapses any previously open row
- Slide-open animation (~150ms height transition) matching MarketsSlot accordion pattern
- Dropdown stays open after clicking an entity to fly-to (user can quickly jump between entities)

### Entity List Content
- Each entity row shows: entity label + one key metric
  - Flights: callsign (or hex if unidentified) + altitude
  - Ships: name/MMSI + speed
  - Events: event type label + Goldstein score
  - Hit Sites: site name + number of attacks
- Per-category proximity sorting:
  - Flights/Events: sorted by distance from Tehran (35.69°N, 51.39°E) primary, Tel Aviv (32.07°N, 34.78°E) as secondary tiebreaker
  - Ships: sorted by distance from Strait of Hormuz (~26.56°N, 56.25°E)
  - Hit Sites: sorted by number of attacks descending (most attacked first)
- Currently selected entity is highlighted with a subtle background accent in the list

### Empty & Zero States
- Counter rows with count = 0 are disabled and non-expandable (dimmed, no chevron)
- All 6 site type rows always visible in Struck Sites section, even at 0
- If an expanded dropdown's count drops to 0 live, the dropdown stays open showing an empty state message ("No entities in view")

### List Density & Scrolling
- Max 8 entity rows visible before scrolling kicks in (~200px max height)
- Thin scrollbar always visible when content overflows (dark theme styled)
- Range-based overflow indicator below the list: "Showing 8–16 of 142" that updates as user scrolls

### Claude's Discretion
- Exact haversine sort implementation details
- Entity row hover state styling
- Scroll position tracking for the range indicator
- Transition easing curve

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `CounterRow.tsx`: Current atomic component (label + value + delta animation) — will be extended with expand/collapse and child list
- `useCounterData.ts`: Returns visibility-aware counts but not entity arrays — needs to also expose filtered entity arrays per subtype
- `useFilteredEntities`: Provides the filtered flight/ship/event arrays that feed the counts
- `selectEntity()` + `openDetailPanel()`: uiStore actions for entity selection and detail panel
- `map.flyTo()` pattern: Used in SearchModal and NotificationBell for animated map navigation
- MarketsSlot accordion: CSS max-height transition pattern for smooth expand/collapse

### Established Patterns
- Zustand `s => s.field` selectors for minimal re-renders
- `useSelectedEntity` for cross-store entity lookup by ID
- Delta animation with `animate-delta` CSS class (3s fade)
- OverlayPanel wrapper for sidebar content sections

### Integration Points
- `CountersSlot.tsx`: Parent component that renders all CounterRow instances — will manage accordion state
- `useCounterData`: Must be extended to return entity arrays alongside counts
- `BaseMap.tsx`: flyTo logic already wired via FlyToHandler pattern
- `uiStore`: selectEntity and openDetailPanel actions already available

</code_context>
