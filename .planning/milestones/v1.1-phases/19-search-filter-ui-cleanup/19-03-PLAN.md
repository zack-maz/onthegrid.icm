**1. Update `src/hooks/useEntityLayers.ts`:**
Add search filter awareness to every entity layer's `getColor` accessor:

- Read from searchStore at the top of the hook: `const isFilterActive = useSearchStore(s => s.isFilterMode && s.matchedIds.size > 0)` and `const matchedIds = useSearchStore(s => s.matchedIds)`.
- Define `SEARCH_DIM_ALPHA = 15` (much dimmer than active entity DIM_ALPHA of 40, per locked decision "highly transparent").
- In each layer's `getColor` function, BEFORE the existing activeId dimming check, add:
  ```
  if (isFilterActive && !matchedIds.has(d.id)) return [r, g, b, SEARCH_DIM_ALPHA];
  ```
  This means: if search filter is active and this entity is NOT in the matched set, render nearly invisible.
- Update `updateTriggers.getColor` arrays to include `isFilterActive` and `matchedIds` (use `matchedIds.size` as a trigger value since Set reference changes on each update).

- For glow and highlight layers: when isFilterActive, only render glow/highlight for entities in matchedIds. Simplest approach: set `visible: !isFilterActive` on glow/highlight layers, OR filter their data arrays to only include matched entities.

**2. Update `src/components/map/EntityTooltip.tsx`:**
When search filter is active (`isFilterActive`), suppress hover tooltips for non-matching entities:
- Read `isFilterActive` and `matchedIds` from searchStore
- In the hover handler or tooltip render logic: if `isFilterActive && hoveredEntity && !matchedIds.has(hoveredEntity.id)`, do not show tooltip
- Non-matching entities should still be clickable (onClick still works to select + open detail panel), but hover tooltip is suppressed

**3. Update `src/hooks/useSearchResults.ts`:**
Add a useEffect that re-computes matchedIds when entity data changes WHILE filter mode is active (Pitfall 4 fix):
- Watch isFilterMode and query from searchStore
- Watch entity arrays from stores (flights, ships, events, sites)
- When isFilterMode is true AND entity arrays change, re-run search with current query and update matchedIds
- This ensures entities that appear/disappear during polling stay correctly dimmed
- Use a ref to avoid re-running when only the hook re-renders
