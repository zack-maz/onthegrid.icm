## Decisions Made
- SearchResult interface duplicated in useSearchResults.ts rather than re-exporting from searchUtils (cleaner separation between old substring and new evaluator approach)
- Triple-ref pattern (syncSourceRef + prevQueryRef + prevTogglesRef) chosen for loop prevention -- more robust than single ref for async React effects
- Text-only queries (like "iran") don't alter sidebar toggles -- avoids unexpected toggle changes during freetext search
- Sidebar -> search sync only fires when query is non-empty -- prevents generating "type:flight type:ship" query from default toggle state
- buildASTFromToggles uses first type value per group (e.g., 'ground_combat' for showGroundCombat) to keep query concise rather than listing all 8 sub-types
