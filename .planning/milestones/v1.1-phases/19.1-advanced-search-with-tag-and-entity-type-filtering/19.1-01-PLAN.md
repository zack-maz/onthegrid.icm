  <name>Task 1: Query Parser - Tokenizer + Recursive Descent AST Builder</name>
  <files>src/lib/queryParser.ts, src/lib/queryParser.test.ts</files>
  <behavior>
    Tokenizer:
    - "type:flight" -> [{type: 'TAG', prefix: 'type', tagValue: 'flight', negated: false}]
    - "!country:iran" -> [{type: 'TAG', prefix: 'country', tagValue: 'iran', negated: true}]
    - "type:flight AND country:iran" -> [TAG, AND, TAG]
    - "type:flight country:iran" -> [TAG, TAG] (implicit AND)
    - "(a OR b)" -> [LPAREN, TAG/TEXT, OR, TAG/TEXT, RPAREN]
    - "boeing 747" -> [{type: 'TEXT', value: 'boeing'}, {type: 'TEXT', value: '747'}]
    - "!boeing" -> [{type: 'TEXT', value: 'boeing', negated: true}] (negated freeform text)
    - Tokens include start/end positions for syntax highlighting
    - Range operators in tag values preserved: "altitude:>30000" -> prefix:'altitude', tagValue:'>30000'

    Parser (recursive descent):
    - Grammar: expr = term (OR term)*, term = factor (AND|implicit factor)*, factor = '!' factor | '(' expr ')' | tag | text
    - Standard boolean precedence: AND tighter than OR. So `a AND b OR c` = `(a AND b) OR c`.
    - Empty input -> null
    - Single tag -> TagNode
    - "type:flight AND country:iran" -> AndNode(TagNode, TagNode)
    - "type:flight OR type:ship" -> OrNode(TagNode, TagNode)
    - "(type:flight OR type:ship) AND country:iran" -> AndNode(OrNode(TagNode, TagNode), TagNode)
    - "!country:iran" -> TagNode with negated=true
    - "boeing" -> TextNode('boeing')
    - "!boeing" -> NotNode(TextNode('boeing')) -- negated freeform text
    - "boeing 747" -> AndNode(TextNode('boeing'), TextNode('747'))
    - Multiple implicit-AND terms chain correctly
  </behavior>
  <action>
    Create `src/lib/queryParser.ts` with:

    1. QueryNode discriminated union type:
       - `{ type: 'tag'; prefix: string; value: string; negated: boolean }` -- a single tag like `country:iran`
       - `{ type: 'text'; value: string }` -- freeform substring
       - `{ type: 'and'; left: QueryNode; right: QueryNode }`
       - `{ type: 'or'; left: QueryNode; right: QueryNode }`
       - `{ type: 'not'; child: QueryNode }` -- for negated freeform text (!boeing)

    2. Token type with `type: TokenType`, `value: string`, `prefix?: string`, `tagValue?: string`, `negated?: boolean`, `start: number`, `end: number`

    3. `tokenize(input: string): Token[]` -- lexer that handles:
       - Whitespace skipping
       - Parentheses as single-char tokens
       - Words: check for AND/OR keywords (uppercase only), then check for `!` negation prefix, then check for `:` to distinguish TAG from TEXT
       - TAG tokens: split on first `:`, prefix before, tagValue after. Range ops like `>30000` stay in tagValue.
       - Position tracking (start/end) for every token

    4. `parse(input: string): QueryNode | null` -- recursive descent:
       - Standard boolean precedence: AND tighter than OR
       - `parseOr()` -> chains `parseAnd()` with OR
       - `parseAnd()` -> chains `parseFactor()` with AND or implicit AND (two adjacent non-operator tokens)
       - `parseFactor()` -> handles `!` negation (for both tags and freeform text), parenthesized expressions, TAG nodes, TEXT nodes
       - For negated TEXT: `!boeing` produces `{ type: 'not', child: { type: 'text', value: 'boeing' } }`
       - Returns null for empty input

    Handle edge cases: trailing operators ignored, unclosed parens treated as if closed, empty tag value (e.g. `type:`) treated as TEXT.
  </action>

  <name>Task 2: Query Evaluator - AST Walker Against Entity Data</name>
  <files>src/lib/queryEvaluator.ts, src/lib/queryEvaluator.test.ts</files>
  <behavior>
    evaluateQuery(node, entity, context):
    - AND node: both children must match
    - OR node: either child must match
    - NOT node: inverts match of child
    - Tag with negated=true: inverts match
    - TEXT node: substring match against getSearchableFields (backward compat)
    - null node: matches everything (empty query)

    evaluateTag(entity, prefix, value):
    - type: -> entity.type === value (or entity.siteType for site subtypes)
    - country: -> flight originCountry, event actor1/actor2 country match
    - callsign: -> flight label substring match
    - icao: -> flight data.icao24 match
    - mmsi: -> ship data.mmsi match
    - shipname: -> ship data.shipName substring match
    - actor: -> event data.actor1/actor2 substring match
    - location: -> event data.locationName / site label substring match
    - cameo: -> event data.cameoCode match
    - altitude: -> flight data.altitude with range operators (>30000, <10000, exact)
    - speed: -> flight velocity or ship speedOverGround with range operators
    - heading: -> ship data.trueHeading with range operators
    - goldstein: -> event data.goldsteinScale with range operators
    - mentions: -> event data.numMentions with range operators
    - ground: -> flight data.onGround boolean
    - vertical: -> flight verticalRate sign ("climbing" if >0, "descending" if <0)
    - unidentified: -> flight data.unidentified boolean
    - severity: -> computeSeverityScore thresholds (high>50, medium>15, low<=15)
    - near: -> haversine distance to named site <= 50km
    - site: -> site siteType match (nuclear, naval, etc.)
    - status: -> site attack status (healthy/attacked) via computeAttackStatus
    - since: -> entity.timestamp >= parsed relative/absolute time
    - before: -> entity.timestamp <= parsed time
    - date: -> event timestamp date match
    - has: -> attribute presence check (has:callsign -> callsign non-empty)
    - squawk: -> no-op (always true), data not in FlightEntity yet

    Range operator parsing in tag values:
    - ">30000" -> greater than 30000
    - "<10000" -> less than 10000
    - ">=500" -> greater or equal
    - "<=200" -> less or equal
    - "30000" -> exact match
    - "30000-40000" -> range between (inclusive)

    Temporal parsing for since:/before:
    - Relative: "6h", "30m", "2d", "1w" -> subtract from now
    - Absolute: "2026-03-20" -> parse as date
  </behavior>

    2. `evaluateQuery(node: QueryNode | null, entity: MapEntity | SiteEntity, context: EvaluationContext): boolean`

       - AND: left && right
       - OR: left || right
       - NOT: !evaluateQuery(child, ...)
       - Tag with negated=true: !evaluateTag(...)

    4. Pure helper functions: `parseRangeValue(value: string): { op: 'eq'|'gt'|'lt'|'gte'|'lte'|'range'; num: number; num2?: number }`, `parseTemporalValue(value: string, now: number): number` (returns Unix ms).

    serialize(node):
    - TagNode -> "prefix:value" or "!prefix:value" if negated
    - TextNode -> "value"
    - NotNode -> "!child" (for negated freeform text)

    - OrNode -> "left OR right"
    - Parentheses added around OR nodes when inside AND to preserve precedence
    - null -> ""

    Round-trip: parse(serialize(parse(input))) produces equivalent AST to parse(input)
  </behavior>
