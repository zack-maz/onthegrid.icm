<objective>
Create the centralized tag registry that maps every tag prefix to its metadata, value extractors, and display colors.

Purpose: The tag registry is the single source of truth for the tag vocabulary. It serves three consumers: (1) the autocomplete dropdown needs value extractors to suggest known values from live entity data, (2) the syntax highlighting overlay needs per-prefix colors, (3) validation needs to know which prefixes are valid. Building this as a standalone module keeps it testable and avoids scattering tag metadata across components.

Output: A single tagRegistry module with comprehensive metadata for all ~25 tag prefixes.
</objective>

       ```typescript
       interface TagDefinition {
         prefix: string;
         label: string;           // Human display name
         description: string;     // For cheat sheet
         color: string;           // Tailwind text color class for syntax highlighting
         entityTypes: ('flight' | 'ship' | 'event' | 'site')[];  // Which entity categories
         examples: string[];      // Example queries for cheat sheet
         getValues?: (data: EntityDataSources) => TagValue[];  // For autocomplete Stage 2
       }
       ```

    2. Define `EntityDataSources` interface (passed to getValues):
       ```typescript
       interface EntityDataSources {
         flights: FlightEntity[];
         ships: ShipEntity[];
         events: ConflictEventEntity[];
         sites: SiteEntity[];
       }
       ```

    3. Define `TagValue` type: `{ value: string; count: number; label?: string }`

    4. Build `TAG_REGISTRY: Record<string, TagDefinition>` with all ~25 prefixes:
       - Cross-entity tags: type, country, actor, location, severity, near, since, before, has
       - Flight: callsign, icao, altitude, speed, squawk, ground, vertical, unidentified
       - Ship: mmsi, heading, shipname (speed shared with flight)
       - Event: cameo, goldstein, mentions, date
       - Site: site, status

    5. Colors (Tailwind classes):
       - type: text-blue-400
       - country/actor/location: text-emerald-400
       - severity: text-red-400
       - near: text-amber-400
       - since/before/date: text-purple-400
       - flight-specific (callsign, icao, altitude, speed, etc.): text-yellow-400
       - ship-specific (mmsi, shipname, heading): text-violet-400
       - event-specific (cameo, goldstein, mentions): text-orange-400
       - site-specific (site, status): text-green-400
       - has: text-cyan-400

    6. Value extractors (getValues):
       - type: returns all EntityType values + 'site' with counts from entity arrays
       - country: unique originCountry from flights + actor countries from events, with counts
       - actor: unique actor1 + actor2 from events, with counts (deduplicated)
       - location: unique locationName from events + labels from sites
       - site: SiteType values with counts
       - severity: static ["high", "medium", "low"]
       - ground: static ["true", "false"]
       - vertical: static ["climbing", "descending"]
       - unidentified: static ["true", "false"]
       - status: static ["healthy", "attacked"]
       - Other tags with numeric/freeform values: no getValues (undefined)
       - squawk: no getValues (data unavailable)

    7. Export helper functions:
       - `isValidPrefix(prefix: string): boolean` -- checks TAG_REGISTRY
       - `getTagColor(prefix: string): string` -- returns color class or default
       - `getTagValues(prefix: string, data: EntityDataSources): TagValue[]` -- calls registry getValues
       - `ALL_PREFIXES: string[]` -- sorted list of all valid prefixes
