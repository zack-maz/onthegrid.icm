  <name>Task 1: Add in-memory fallback to redis.ts and create rich health endpoint</name>
  <files>server/cache/redis.ts, server/routes/health.ts, server/__tests__/redis-cache.test.ts, server/__tests__/routes/health.test.ts</files>
  <behavior>
    - cacheGetSafe returns cached data from Redis on success AND populates memCache
    - cacheGetSafe returns memCache data with degraded:true when Redis throws
    - cacheGetSafe returns null when both Redis and memCache miss
    - cacheSetSafe writes to both Redis and memCache
    - GET /health returns JSON with: status (ok|degraded), redis (boolean), uptime, timestamp, latencyMs, sources (per-source lastFresh timestamps)
    - GET /health returns status:degraded when redis.ping() fails
  </behavior>
  <action>
    1. Update `server/cache/redis.ts`:
       - Add module-level `const memCache = new Map<string, { data: unknown; fetchedAt: number }>()` as in-memory LRU fallback
       - Add `degraded?: boolean` to CacheResponse type in `server/types.ts`
       - Export `cacheGetSafe<T>(key, logicalTtlMs)`:
         - Try `cacheGet<T>()` -- on success, write to memCache and return result
         - On catch: check memCache for key. If found, return `{ data, stale: true, lastFresh: mem.fetchedAt, degraded: true }`
         - If neither, return null
       - Export `cacheSetSafe<T>(key, data, redisTtlSec)`:
         - Try `cacheSet<T>()` -- always write to memCache regardless of Redis success
         - On catch: swallow error (memCache is already updated)
       - Keep existing `cacheGet`/`cacheSet` unchanged for backward compat (routes can migrate incrementally)

    2. Create `server/routes/health.ts`:
       - Import `redis` from `../cache/redis.js`
       - Import `cacheGet` from `../cache/redis.js`
       - Export `healthRouter` (Express Router)
       - GET / handler:
         - Ping Redis with try/catch, record latency
         - For each source key (flights:adsblol, ships:ais, events:gdelt, news:gdelt, markets:yahoo:1d, weather:open-meteo, sites:v2), attempt `cacheGet` to extract `lastFresh` timestamp. Use a very large logicalTtlMs (999999999) to just get the entry without marking stale.
         - Estimate daily Redis commands: rough formula based on known polling intervals (flights ~2880/day, ships ~2880, events ~96, news ~96, markets ~1440, weather ~48, sites ~1, sources ~100 = ~7541 commands/day * 2 per command = ~15K/day). Include this as `estimatedDailyCommands` field.
         - Return JSON: `{ status, redis, uptime: process.uptime(), timestamp, latencyMs, sources: { flights: lastFresh, ships: lastFresh, ... }, estimatedDailyCommands }`

    3. Update `server/__tests__/redis-cache.test.ts`:
       - Add tests for cacheGetSafe: Redis success path, Redis failure with memCache hit (returns degraded:true), Redis failure with memCache miss (returns null)
       - Add tests for cacheSetSafe: writes to both Redis and memCache, survives Redis failure

    4. Create `server/__tests__/routes/health.test.ts`:
       - Mock redis module
       - Test: returns status ok + redis true when ping succeeds
       - Test: returns status degraded + redis false when ping throws
       - Test: includes sources object with timestamp numbers
  </action>
