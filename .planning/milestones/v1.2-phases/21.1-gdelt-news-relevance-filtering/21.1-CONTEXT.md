# Phase 21.1: GDELT News Relevance Filtering - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Reduce false positive conflict news by overhauling the server-side news filtering pipeline. Replace the current binary keyword pass/fail with an NLP-based relevance scoring system that extracts actor-action-target triples from headlines and summaries, leverages GDELT metadata fields, and assigns a confidence score to each article. Articles below a configurable threshold are excluded. Does NOT change the notification center, news clustering, or client-side rendering (those consume the improved data).

</domain>

<decisions>
## Implementation Decisions

### Keyword reclassification
- **Strict minimum non-ambiguous set**: only `airstrike`, `missile`, `bombing`, `shelling`, `casualties`, `invasion`, `drone` remain non-ambiguous (pass on their own)
- **All geographic terms** (tehran, baghdad, iran, israel, etc.) move to AMBIGUOUS — require a co-occurring non-ambiguous term
- **All diplomatic/political terms** (sanctions, negotiations, ceasefire, tensions, etc.) move to AMBIGUOUS
- **All organization names** (IRGC, Hezbollah, Hamas, Houthi, Pentagon, CENTCOM, NATO, IDF, Mossad) move to AMBIGUOUS
- **Expanded exclusion patterns** — Claude's discretion on what entertainment/historical/educational patterns to add beyond current sports/celebrations/finance set

### NLP-based headline semantics
- Use a **lightweight NLP library** (compromise, wink-nlp, or similar) for part-of-speech tagging and entity extraction
- **Extract actor-action-target triples** from article text — e.g., "Iran launched missile at Israel" → `{actor: "Iran", action: "missile launch", target: "Israel"}`
- Apply extraction to **both titles and summaries** (not just titles)
- **Store and expose triples** in the API response — add `actor`, `action`, `target` fields to `NewsArticle` type for downstream consumption (notifications, detail panel)

### Confidence scoring (replaces binary filter)
- **Replace the binary keyword filter** with a 0-1 relevance score
- **Three scoring factors:**
  1. **NLP triple completeness** — full actor-action-target = high score; partial extraction = medium; no actionable triple = low
  2. **GDELT tone score** — strongly negative tone correlates with conflict reporting; feed into score calculation
  3. **Source reliability weighting** — major outlets (BBC, Reuters, Al Jazeera) weighted higher than unknown GDELT sources
- Note: keyword density/tier explicitly NOT included as a scoring factor (user chose NLP-first approach)
- **Threshold: 0.7** (strict) — configurable via `NEWS_RELEVANCE_THRESHOLD` env var with 0.7 default
- **Score display** — visible in dev mode only (consistent with NotificationCard severity score pattern), hidden in production

### GDELT metadata leverage
- **Tone** — feed GDELT's pre-computed tone score (-10 to +10) into relevance scoring formula
- **V2Themes** — boost + negative gate approach:
  - Conflict themes (MILITARY, ARMED_CONFLICT, KILL, etc.) boost the relevance score
  - If article ONLY has non-conflict themes (ECON, EDUCATION, SPORTS), hard reject regardless of other signals
- **V2Locations** — merge approach: use GDELT's V2Locations lat/lng as primary location source; fall back to keyword-based geo matching for articles where GDELT didn't extract a location (RSS articles, etc.)
- **V2Counts** — ranking only; does not affect whether an article passes the threshold; used to sort/rank within the feed (higher mentions = more prominent)

### Claude's Discretion
- NLP library choice (compromise, wink-nlp, or other lightweight option suitable for serverless)
- Exact exclusion patterns to add beyond current set
- Relevance score formula weights (how to balance triple completeness vs tone vs source reliability)
- Conflict-related V2Theme code list (which GDELT theme codes count as "conflict-related")
- Non-conflict V2Theme code list for the hard reject gate
- Source reliability tiers and their exact weight values

</decisions>

<specifics>
## Specific Ideas

- The core motivation: "Tehran condemns airstrike in Yemen" should NOT drop a marker on Tehran — the current keyword filter treats this as a conflict event in Tehran because both "Tehran" and "airstrike" match
- Actor-action-target triples enable richer notification display (e.g., "Iran → missile → Israel" instead of just a headline)
- V2Locations from GDELT uses NLP for geo extraction — more reliable than keyword-based matching for determining WHERE a conflict event actually occurred
- The 0.7 threshold is deliberately strict — user prefers missing some articles over showing false positives ("numbers over narratives" project philosophy)

</specifics>
