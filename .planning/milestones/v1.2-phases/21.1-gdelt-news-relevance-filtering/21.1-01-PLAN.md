    - Test: score is clamped to 0-1 range

  <action>
    1. Create `server/lib/relevanceScorer.ts`:
       - Import `ArticleTriple` from `./nlpExtractor.js`
       - Import `nlp` from `compromise` for negativity analysis

       - Export `ScoringInput` interface:
         ```typescript
         interface ScoringInput {
           triple: ArticleTriple;
           source: string;        // "BBC", "GDELT", "Al Jazeera", etc.
           domain?: string;       // GDELT domain for reliability lookup
           title: string;
           summary?: string;
         }
         ```

       - Define `SOURCE_RELIABILITY` map (exported for testing):
         - Tier 1 (1.0): bbc.co.uk, bbc.com, reuters.com, apnews.com, afp.com
         - Tier 2 (0.9): aljazeera.com, timesofisrael.com, middleeasteye.net, haaretz.com
         - Tier 3 (0.8): tehrantimes.com, irna.ir, presstv.ir
         - Tier 4 default (0.6): any unknown domain/source
         - Lookup by source name first (RSS feeds), then by domain (GDELT articles)

       - Define `CONFLICT_VERBS` set: strike, kill, bomb, destroy, attack, shell, invade, launch, fire, shoot, target, hit, blast, detonate, intercept, deploy, seize, capture, raid, assassinate, wound, shatter, demolish, obliterate

       - Define `EXCLUSION_PATTERNS` array (expanded per research): include current patterns plus historical/documentary, entertainment, sports expanded, education/academic, technology, weather/natural patterns from RESEARCH.md

       - Export `computeRelevanceScore(input: ScoringInput): number`:
         a. **Pre-check exclusion patterns** — if title+summary matches any exclusion pattern, return 0 immediately
         b. **Factor 1: NLP Triple Completeness (0-0.45)**
            - actor present: +0.12
            - action present: +0.20 (weighted highest -- most informative)
            - target present: +0.13
            - Full triple = 0.45, partial = proportional
         c. **Factor 2: Negativity/Conflict Signal (0-0.35)**
            - Use compromise to extract verbs from title+summary
            - Count how many verbs are in CONFLICT_VERBS set
            - conflictVerbRatio = conflictVerbCount / max(totalVerbCount, 1)
            - negativityScore = conflictVerbRatio * 0.35
            - (This replaces the GDELT tone score per research adaptation)
         d. **Factor 3: Source Reliability (0-0.20)**
            - Look up source in SOURCE_RELIABILITY map
            - sourceScore = reliability * 0.20
         e. Return Math.min(1, factor1 + factor2 + factor3)

    2. Create `server/__tests__/lib/relevanceScorer.test.ts`:
       - Test the behaviors listed above
       - Use `@vitest-environment node` pragma
       - Import from `../../lib/relevanceScorer.js`

         - "Celebrity gossip roundup" should score near 0
