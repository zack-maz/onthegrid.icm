<task type="auto">
  <name>Task 1: Execute k6 load test and Playwright browser validation against production</name>
  <files>scripts/load-test.js</files>
  <action>
**Pre-flight checks:**
1. Verify k6 is installed: `k6 version`. If not installed, run `brew install k6`.
2. Verify Playwright is available: `npx playwright --version`. If browsers not installed, run `npx playwright install chromium`.
3. Run a quick smoke test against production first to confirm endpoints are alive: `npx tsx scripts/smoke-test.ts https://irt-monitoring.vercel.app`

**Execute load test:**
1. Create `load-test-results/` directory if it doesn't exist.

2. Start Playwright browser validation in the background:
   ```
   npx playwright test scripts/load-test.spec.ts --reporter=list 2>&1 | tee load-test-results/playwright-output.txt &
   ```

3. Run k6 load test (5-minute duration):
   ```
   k6 run scripts/load-test.js --out json=load-test-results/k6-results.json 2>&1 | tee load-test-results/k6-output.txt
   ```

4. Wait for both to complete.

**Analyze results:**
- Parse k6 output for: p95 latency per endpoint, error rate, 429 count, cache hit ratio, cold start latencies
- Parse Playwright output for: pass/fail status, any JS errors captured, screenshots taken
- Check if k6 thresholds passed or failed
- Compare health endpoint snapshots (setup vs teardown) for Redis status changes

**If k6 script needs minor adjustments** (e.g., threshold too strict, scenario timing off, import issues with k6 runtime): fix the script and re-run. k6 has its own JS runtime, so if any Node.js-isms crept in during Plan 01, fix them here.

**If rate limits need minor adjustments** based on results (e.g., a limit is too restrictive for legitimate multi-user use): adjust in `server/middleware/rateLimit.ts`. Only make changes if the data clearly shows a limit is too low for the intended multi-user scenario.

**Capture key metrics** for the SUMMARY:
- Total requests made
- p50/p95/p99 latency per endpoint
- 429 rate per endpoint
- Cache HIT ratio
- Redis health before/after
- Any 5xx errors
- Playwright test pass/fail
  </action>
  <verify>
    <automated>test -f load-test-results/k6-output.txt && echo "k6 results captured" || echo "MISSING: k6 output"; test -f load-test-results/playwright-output.txt && echo "playwright results captured" || echo "MISSING: playwright output"</automated>
  </verify>
  <done>k6 load test completed against production with results captured. Playwright browser validation completed. Key metrics extracted: p95 latency, error rate, 429 count, cache hit ratio, Redis health delta.</done>
</task>
