# Phase 20.1: Geographical & Weather Layers - Context

**Gathered:** 2026-03-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Two stackable visualization overlays on the existing Dark Matter basemap: (1) a monochrome terrain elevation tinting with contour lines and geographic feature labels, and (2) a real-time weather layer with temperature heatmap and wind barbs from Open-Meteo API. Both plug into the layer architecture built in Phase 20 (layerStore, MapLegend, LayerTogglesSlot).

</domain>

<decisions>
## Implementation Decisions

### Terrain Overlay Intensity
- Subtle wash at 20-30% opacity — adds depth without competing with entity layers
- Mountains visible as light blue-gray hints, plains near-invisible
- Entities render clearly on top; basemap labels/borders stay readable
- 3D terrain exaggeration stays at 3.0x regardless of geographic layer toggle — layer adds color/contours only, doesn't modify terrain mesh

### Elevation Color Palette
- Cool blue-gray tint (not pure grayscale) — night-vision topographic feel
- Low elevation: #1a1a2e (near-invisible), Mid: #334155 (slate blue-gray), High: #94a3b8 (light slate)
- Blends naturally with Dark Matter basemap and distinguishes from plain gray UI elements

### Contour Lines
- Accent lines only — thin, low-opacity, zoom-dependent
- Not visible at overview (z4-6), faint lines appear at regional (z7-8), clear detail at local (z9+)
- White/gray at 15-20% opacity, ~500m elevation step interval

### Geographic Feature Labels
- Four categories: mountain ranges, deserts & plains, water bodies & straits, major rivers
- Subtle cartographic text directly on the map — uppercase, letter-spaced, 50% opacity, no background
- Uniform styling across all categories (no color-coding by type)
- Zoom-dependent tiers:
  - z4-5: Zagros, Persian Gulf, Caspian Sea (largest features)
  - z6-7: + Dasht-e Kavir, Elburz, Red Sea, Dasht-e Lut, Syrian Desert
  - z8+: + Tigris, Euphrates, Strait of Hormuz, Bab el-Mandeb, Karun
- ~15 labels total across 3 zoom tiers

### Weather Data Scope
- Temperature AND wind data from Open-Meteo (both free, no API key)
- Temperature as smooth heatmap (Deck.gl HeatmapLayer with Gaussian interpolation)
- Wind as meteorological barbs (line + flags for speed, rotated for direction)
- Wind speed displayed in knots only (aviation standard)

### Temperature Heatmap
- 1-degree grid (~1,080 data points across 15-42N, 30-70E)
- Smooth interpolated rendering via Deck.gl HeatmapLayer — no visible grid boundaries
- 20-30% opacity matching subtle terrain wash
- Color scale: blue (cold) -> green -> yellow -> red (hot)
- Dual-unit legend (C and F)

### Wind Barbs
- Sparser than temperature grid — every 2-3 degrees (~150-200 barbs)
- Classic meteorological wind barb style (short barb = 10 kn, long barb = 20 kn, triangle = 50 kn)
- Subtle white at low opacity, rotated to show wind direction
- No overlap at default z4.6 overview zoom

### Weather Tooltip
- Hovering over weather overlay shows tooltip with nearest grid point data
- Shows temperature (C/F dual unit) and wind (direction + speed in kn)
- Must coexist with existing EntityTooltip without conflicts

### Server & Caching
- `/api/weather` endpoint with Redis-cached Open-Meteo data
- 30-60 min polling interval (weather changes slowly)
- Cache key: `weather:open-meteo`
- Single Open-Meteo request for entire grid

### Claude's Discretion
- Exact legend positioning and stacking layout
- Contour line rendering approach (MapLibre hillshade vs custom layer)
- Grid resolution for weather temperature sampling within the 1-degree constraint
- Transition animation easing curve for layer fade
- Wind barb icon atlas implementation
- Tooltip priority when weather and entity tooltips overlap
- Which specific geographic features qualify as "major" within each zoom tier (guided by the ~15 total constraint)

</decisions>

<specifics>
## Specific Ideas

- Terrain should feel like a night-vision topographic overlay — cool blue-gray, subtle, informational
- Wind barbs are classic meteorological style, not simplified arrows — this is an intelligence tool, not a consumer weather app
- Geographic labels spread across features like cartographic text (e.g., "Z A G R O S  M O U N T A I N S" letterspaced across the range), not pinpoint labels
- Weather tooltip is a nice-to-have data inspection tool — should feel like hovering over a weather radar

</specifics>
