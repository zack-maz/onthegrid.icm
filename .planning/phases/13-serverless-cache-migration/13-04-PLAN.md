<objective>
Re-introduce GDELT event backfill into the events route to close the UAT gap where the date range filter shows no historical data.

Purpose: Phase 13 Plan 03 removed the old startup backfill mechanism (incompatible with serverless). Without it, the Redis event accumulator only grows by 15-minute GDELT increments and resets on Redis eviction (2.5h TTL). The date range filter UI works but has no historical data to filter against.

Output: Events route with lazy on-demand backfill that seeds historical events on first request when Redis is empty, using the existing `backfillEvents()` function from `server/adapters/gdelt.ts`.
</objective>

Strategy: Lazy on-demand backfill triggered when Redis event cache is empty (first request after cold start or Redis eviction).

1. **Add imports** to `server/routes/events.ts`:
   - Import `backfillEvents` from `../adapters/gdelt.js`
   - Import `redis` from `../cache/redis.js` (for direct backfill timestamp tracking)

2. **Add backfill constants**:
   - `BACKFILL_KEY = 'events:backfill-ts'` — Redis key storing last backfill Unix ms timestamp
   - `BACKFILL_COOLDOWN_MS = 3_600_000` — 1 hour cooldown to prevent hammering GDELT master list
   - `BACKFILL_DAYS` — compute dynamically: `Math.ceil((Date.now() - WAR_START) / 86_400_000)` to cover full war timeline

3. **Add `shouldBackfill()` helper** (async):
   - Check `redis.get<number>(BACKFILL_KEY)`
   - Return `true` if key is null (never backfilled) OR if `Date.now() - lastTs > BACKFILL_COOLDOWN_MS`
   - Return `false` otherwise

4. **Modify the route handler** — in the `cached === null` branch (cache miss), AFTER the normal `fetchEvents()` call:
   - Call `shouldBackfill()`
   - If true: call `backfillEvents(BACKFILL_DAYS)` in a try/catch (non-fatal)
   - Merge backfill results into the eventMap (same Map-based merge-by-ID pattern already in the route)
   - Store `redis.set(BACKFILL_KEY, Date.now(), { ex: REDIS_TTL_SEC })` to record backfill timestamp
   - Log `[events] backfill: merged N historical events`
   - If backfillEvents throws: log warning and continue with just fetchEvents data

5. **Important: Backfill only on cache miss** — When cache exists (fresh or stale), the route already has accumulated data. Backfill is only needed to seed an empty accumulator. This keeps the normal request path fast.

6. **Update tests** in `server/__tests__/routes/events.test.ts`:
