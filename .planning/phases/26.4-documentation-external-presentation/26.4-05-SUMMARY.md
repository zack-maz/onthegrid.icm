## Decisions Made

- **Mermaid inline beats committed SVG/PNG.** Mermaid renders natively on
  GitHub, diffs cleanly in PRs, and evolves with the code. The build-step
  cost is zero.
- **Plain-flowchart fallback in system-context.md.** C4Context is not
  supported by every Mermaid renderer; a second equivalent `flowchart LR`
  block guarantees the diagram still shows something in older viewers.
- **Ontology split into 4 files, not 1.** A single monolithic
  `ontology.md` becomes a wall of text; four focused files (types,
  algorithms, state machines, complexity) are navigable.
- **9 sequence diagrams, not 8.** The plan required 8 minimum; geocode
  was added as a 9th because it's a genuinely distinct flow with a
  two-tier lookup pattern worth documenting.
- **TODO(26.2) tech debt labeled inline.** Hardcoded CAMEO tables,
  centroid dispersion gaps, NLP extraction placeholder fields, and
  coarse basin lookup are labeled where they appear in the diagrams
  rather than hidden in an "issues" section. Honesty is a portfolio
  signal.
- **Source pointers use relative repo paths.** `../../server/types.ts`
  rather than `server/types.ts` or a GitHub URL, so the links work both
  in GitHub rendering and in local editors (VS Code, Obsidian).
- **Class diagram only for `MapEntity`.** It's the single most important
  ontology concept; every other type catalog is a prose section with
  code blocks, which is lighter to read and edit.

## Deviations from Plan

### Plan scope deviations

- **9 sequence diagrams instead of 8.** The plan asked for "at least 8";
  geocode was added as a 9th because it has a distinct two-tier lookup
  pattern (client-side site bbox check → server-side Nominatim) that
  deserved its own diagram. Still within scope — "at least 8" was a
  floor.
- **Post-commit fix for SWR math wording.** Prettier mangled the
  expression "s-maxage + swr" inside backticks into "s-maxage - swr",
  inverting the meaning. Reworded to "up to `(s-maxage seconds) + (swr
seconds)` old" so prettier can't mis-format it. Committed separately
  as `c61a437`.
- **Post-push self-check correction for Escape key behavior.** My first
  draft of state-machines.md claimed Escape pops one view from the
  navigation stack. When I verified against
  `src/hooks/useEscapeKeyHandler.ts`, the actual behavior is that
  Escape always closes the detail panel entirely and calls
  `clearStack()` — within-panel back navigation is only the breadcrumb
  arrow. Corrected in `350af36`.

### No deviation rule 1-4 fires

- **No Rule 1 (auto-fix bugs).** The task is documentation-only; no
  behavior was changed. The SWR-math and Escape-key fixes are doc
  corrections, not code fixes.
- **No Rule 2 (missing critical).** No test-writing or missing
  validation involved.
- **No Rule 3 (blocking issues).** The `docs/architecture/` directory
  didn't exist yet; `mkdir -p` handled that without incident.
- **No Rule 4 (architectural decisions).** Scope was clear from the
  plan frontmatter and context.
