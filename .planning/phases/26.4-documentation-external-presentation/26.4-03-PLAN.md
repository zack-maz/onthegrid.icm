  <action>
1. Create server/schemas/cacheResponse.ts with a generic helper:
   import { z, ZodTypeAny } from 'zod'
   export function cacheResponseSchema<S extends ZodTypeAny>(dataSchema: S) {
     return z.object({
       data: dataSchema,
       stale: z.boolean(),
       lastFresh: z.number(),
       rateLimited: z.boolean().optional(),
       degraded: z.boolean().optional(),
     })
   }
   This mirrors the OpenAPI CacheResponse<T> wrapper from server/openapi.yaml allOf composition.

2. Create server/middleware/validateResponse.ts with sendValidated helper:
   - import type { Response } from 'express'
   - import type { ZodTypeAny, z } from 'zod'
   - import { logger } from '../lib/logger.js'
   - import { AppError } from './errorHandler.js'
   - export function sendValidated<S extends ZodTypeAny>(res: Response, schema: S, payload: unknown): void
   - Inside: const parsed = schema.safeParse(payload)
   - If !parsed.success: in dev (NODE_ENV !== 'production') throw new AppError(500, 'RESPONSE_SCHEMA_MISMATCH', JSON.stringify(parsed.error.issues)); in prod logger.warn({ issues, path: res.req.path }, 'response schema mismatch') and res.json(payload)
   - Else: res.json(parsed.data as z.infer<S>)
   See research §"Pattern 3: Zod Response Schema Validation" Option A example for the exact pattern.

3. Wire sendValidated into 3 routes (Claude's discretion which 3 — recommendation: flights, events, water):

   For server/routes/flights.ts:

   - Define const flightsResponseSchema = cacheResponseSchema(z.array(flightEntitySchema))
   - Replace res.json(result) with sendValidated(res, flightsResponseSchema, result)
   - Import sendValidated from ../middleware/validateResponse.js
   - Import cacheResponseSchema from ../schemas/cacheResponse.js

   For server/routes/events.ts: same pattern with eventEntitySchema
   For server/routes/water.ts: same pattern with waterFacilityEntitySchema (or whatever the data shape is)
