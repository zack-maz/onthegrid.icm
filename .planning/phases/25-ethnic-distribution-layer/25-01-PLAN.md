<task type="auto">
  <name>Task 2: GeoEPR extraction script with overlap detection and ethnic-zones.json data asset</name>
  <files>scripts/extract-ethnic-data.ts, src/data/ethnic-zones.json</files>
  <action>
Create `scripts/extract-ethnic-data.ts` following the pattern of `scripts/extract-geo-data.ts`:

1. **Download GeoEPR-2021.geojson** from `https://icr.ethz.ch/data/epr/geoepr/` (direct GeoJSON download). If direct URL fails (403/404), try Harvard Dataverse mirror. Log download progress.

2. **Filter to Middle East bbox**: { latMin: 0, latMax: 50, lngMin: 20, lngMax: 80 }. Reuse the overlapsFilterRegion pattern (walk coordinates, compute bbox, check overlap).

3. **Name-mapping table** from GeoEPR group names to our 10-group taxonomy:
   - First pass: dump ALL unique group names found in Middle East bbox to console
   - Map known names case-insensitively: Kurds/Kurdish -> kurdish, Arabs/Arab -> arab, Persians/Persian -> persian, Baluch/Baluchis/Baloch -> baloch, Turkmen/Turkmens -> turkmen, Druze/Druse -> druze, Alawites/Alawite/Alawi -> alawite, Yazidis/Yazidi -> yazidi, Assyrians/Assyrian/Chaldo-Assyrians -> assyrian, Pashtuns/Pashtun/Pathans -> pashtun
   - Log unmapped groups (Azeris: omit per CONTEXT.md -- our 10-group list does not include them)

4. **Merge cross-border polygons** per group into single MultiPolygon features. Collect all polygon rings from all country-specific features into one MultiPolygon coordinates array per group. Do NOT require @turf/union -- simple coordinate array concatenation suffices (visual rendering, not topological analysis).

5. **Detect overlap zones**: After merging, compute bounding boxes for each group's merged polygon. Where two or more group bounding boxes overlap AND their actual coordinates suggest co-location (within same 0.5-degree grid cell), create overlap features:
   - For each grid cell (0.5 degree), count which merged groups have coordinates in that cell
   - Where 2+ groups share a cell, mark that geographic region as an overlap zone
   - Create overlap features as Polygon/MultiPolygon with properties: `{ groups: string[], label: string }` where groups is the sorted array of overlapping EthnicGroup IDs and label is the joined display names (e.g., "Kurdish / Turkmen / Arab")
   - Known overlap areas to look for: Kirkuk (Kurdish + Turkmen + Arab), NE Syria (Kurdish + Arab + Assyrian), Nineveh (Kurdish + Turkmen + Yazidi + Assyrian)

6. **Single-group features** get properties: `{ group: string, label: string }` (singular `group` string)
   **Overlap features** get properties: `{ groups: string[], label: string }` (plural `groups` array)

7. **Round coordinates** to 2 decimal places using roundCoords pattern from extract-geo-data.ts.

8. **Write** to `src/data/ethnic-zones.json`. Log per-group feature counts, overlap zone count, total file size in KB, and any missing groups.

9. **Run the script**: `npx tsx scripts/extract-ethnic-data.ts`

FALLBACK: If GeoEPR download fails entirely or data format is unworkable, construct approximate ethnic zone polygons manually using well-known geographic boundaries from published ethnic maps. Define each of the 10 zones as simple Polygon features with approximate coordinates. For overlap zones (Kirkuk area at minimum), create features with `groups: ['kurdish', 'turkmen', 'arab']` property. Log clearly that fallback was used. Target file size under 100KB with manual polygons.

Output format per single-group feature:
- type: "Feature"
- properties: { group: "kurdish", label: "Kurdish" }
- geometry: { type: "Polygon" or "MultiPolygon", coordinates: [...] }

Output format per overlap feature:
- type: "Feature"
- properties: { groups: ["kurdish", "turkmen", "arab"], label: "Kurdish / Turkmen / Arab" }
- geometry: { type: "Polygon" or "MultiPolygon", coordinates: [...] }
