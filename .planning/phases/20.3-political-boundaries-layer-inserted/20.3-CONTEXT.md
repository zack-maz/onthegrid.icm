# Phase 20.3: Political Boundaries Layer - Context

**Gathered:** 2026-03-23
**Status:** Ready for planning

<domain>
## Phase Boundary

A toggleable visualization overlay that color-codes countries by alliance/faction alignment, renders thick faction-colored borders with diagonal hatching fills, and marks disputed territories with dashed outlines. Uses Natural Earth 110m GeoJSON (static bundle). Plugs into the layer architecture built in Phase 20 (layerStore, MapLegend, LayerTogglesSlot).

</domain>

<decisions>
## Implementation Decisions

### Faction/Alliance Definitions
- Three-axis model: Iran Axis, US-Aligned, Turkic Bloc, plus Contested and Neutral categories
- **Iran Axis**: Iran, Syria, Yemen
- **US-Aligned**: Israel, Saudi Arabia, UAE, Bahrain, Jordan, Kuwait, Qatar
- **Turkic Bloc**: Turkey, Azerbaijan, Turkmenistan
- **Contested** (dual-faction): Iraq (Iran + US), Lebanon (Iran + US), Pakistan (Iran/China + US)
- **Neutral**: Oman, Afghanistan, and remaining countries in the map bounds
- **Egypt**: Claude's discretion — place based on current conflict context

### Faction Colors
- Iran Axis: #ef4444 (red — consistent with existing conflict/threat coloring)
- US-Aligned: #3b82f6 (blue)
- Turkic Bloc: #f59e0b (amber)
- Contested: alternating stripes of the two relevant faction colors
- Neutral: light gray

### Country Fill Style
- Fine diagonal hatching (45-degree lines) filling each country polygon in the faction color
- 30-40% opacity — visible at overview zoom without overpowering entities or threat heatmap
- Contested countries: alternating dual-color diagonal stripes (e.g., Iraq = alternating red/blue lines)
- Neutral countries: light gray hatching at lower opacity (distinguishes "neutral" from "not in dataset")
- No solid polygon fills — hatching pattern only, like a technical drawing

### Border Rendering
- Own GeoJSON borders from Natural Earth 110m dataset (~200KB, static client bundle)
- Thick 3px faction-colored border lines per country
- Same style for all borders (no de-emphasis for same-faction borders)
- CARTO basemap borders (boundary_country_outline, boundary_country_inner) hidden when political layer is active, restored when off
- Imperative toggle via map.setLayoutProperty('visibility', 'none'/'visible') — same pattern as ROAD_LABEL_LAYERS in handleLoad

### Disputed Territories
- 7 disputed zones: Kurdistan, Gaza & West Bank, Golan Heights, Crimea & Eastern Ukraine, Nagorno-Karabakh, Khuzestan, Balochistan
- Visual treatment: thick dashed white outline (2px) around disputed zone boundary
- Zone name label centered inside each territory (uppercase, letter-spaced, matching geographic label style)
- Zoom-dependent visibility:
  - z5+: Large zones (Kurdistan, Crimea, Balochistan)
  - z7+: Smaller zones (Gaza, Golan Heights, Nagorno-Karabakh, Khuzestan)

### Legend
- Faction color key with discrete swatches (not gradient bar) — diverges from LEGEND_REGISTRY gradient pattern
- Title: "Factions"
- Shows: colored dot/swatch + faction name for each group (Iran Axis, US-Aligned, Turkic Bloc, Contested, Neutral)
- Appears in same bottom-left legend area, stacks with other active layer legends

### Data Source
- Natural Earth 110m simplified boundaries (~200KB, gzips to ~50KB)
- Static client bundle (import as JSON asset) — no server route needed
- Same pattern as geoFeatures.ts in Phase 20.1
- Disputed territory boundaries may need separate GeoJSON data or hand-drawn polygons

### Claude's Discretion
- Egypt faction assignment (US-aligned vs neutral based on conflict context)
- Exact hatching line spacing and stroke width within the 30-40% opacity constraint
- Hatching implementation approach (SVG pattern fill, canvas, or MapLibre fill-pattern)
- Disputed territory GeoJSON sourcing (Natural Earth admin-1 boundaries vs hand-drawn)
- How to implement the discrete-swatch legend variant within or alongside LEGEND_REGISTRY
- Layer stacking position relative to existing layers (likely between geographic and threat)
- Transition animation for CARTO border show/hide

</decisions>

<specifics>
## Specific Ideas

- Hatching should feel like a military/intelligence technical drawing — diagonal lines, not solid fills
- "Thick and can't miss it" — borders should be unmistakable when the layer is active
- Contested countries with alternating faction-color stripes immediately communicate "split loyalty"
- Neutral countries get light gray hatching to distinguish from "outside the dataset" (no hatching)
- Layer is informational context about geopolitical alignment, complementing the threat heatmap's operational data

</specifics>
