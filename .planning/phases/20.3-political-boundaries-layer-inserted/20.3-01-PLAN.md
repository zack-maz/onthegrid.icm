**politicalData.ts** -- Create the data module following geoFeatures.ts pattern (static GeoJSON exports):

1. Define `FactionId` type: `'iran' | 'us' | 'turkic' | 'contested_iran_us' | 'contested_iran_china_us' | 'neutral'`

2. Define `FACTION_MAP: Record<string, FactionId>` mapping ADM0_A3 codes to factions (use ADM0_A3, NOT ISO_A3 -- some countries have `-99` for ISO_A3):
   - Iran Axis: `IRN`, `SYR`, `YEM`
   - US-Aligned: `ISR`, `SAU`, `ARE`, `BHR`, `JOR`, `KWT`, `QAT`, `EGY` (Egypt placed US-aligned per Camp David Accords / US military aid)
   - Turkic Bloc: `TUR`, `AZE`, `TKM`
   - Contested (Iran+US): `IRQ`, `LBN`
   - Contested (Iran/China+US): `PAK`
   - Neutral: `OMN`, `AFG` (plus all remaining countries within MAX_BOUNDS default to neutral)

3. Define `FACTION_COLORS: Record<FactionId, [number, number, number]>`:
   - iran: [239, 68, 68] (#ef4444)
   - us: [59, 130, 246] (#3b82f6)
   - turkic: [245, 158, 11] (#f59e0b)
   - contested_iran_us: uses dual pattern (red+blue)
   - contested_iran_china_us: uses dual pattern (red+blue, simplified)
   - neutral: [156, 163, 175] (#9ca3af)

4. Define `FACTIONS` array with objects: `{ id: FactionId, isoCodes: string[], patternName: string, color: string }` -- one entry per faction for iterating fill layers. The contested factions get their own entries. Export this for PoliticalOverlay to iterate.

5. Define `CONTESTED_COLOR_PAIRS: Record<string, [[number,number,number], [number,number,number]]>`:
   - `contested_iran_us`: [[239,68,68], [59,130,246]]
   - `contested_iran_china_us`: [[239,68,68], [59,130,246]]

6. Build `COUNTRIES_GEOJSON` as a `FeatureCollection<Polygon | MultiPolygon>`. Since Natural Earth 110m is ~500KB raw and we need to pre-filter:
   - Download Natural Earth 110m countries GeoJSON from `https://raw.githubusercontent.com/nvkelso/natural-earth-vector/master/geojson/ne_110m_admin_0_countries.geojson` and extract ONLY the countries within the map's approximate bounds (lat 0-50, lng 20-80).
   - Strip all properties except `ADM0_A3` and `NAME` to minimize bundle size.
   - Since we cannot fetch at runtime, define the GeoJSON inline as a typed constant. Include all countries whose polygons intersect the bounding box [20, 0, 80, 50]. This should cover approximately 30-35 countries. For each feature, keep only `{ ADM0_A3: string; NAME: string }` properties plus geometry coordinates.
   - IMPORTANT: This file will be large (~100-200 lines of coordinate data). That is expected for inline GeoJSON.

7. Build `DISPUTED_GEOJSON` as a `FeatureCollection<Polygon>` with 7 hand-drawn zones:
   - Kurdistan (z5+): approximate polygon covering ~[43,34]-[46,38] area
   - Crimea & Eastern Ukraine (z5+): ~[33,44.4]-[36.5,46.2]
   - Balochistan (z5+): ~[58,25]-[66,30]
   - Gaza & West Bank (z7+): ~[34.2,31.2]-[35.6,32.6]
   - Golan Heights (z7+): ~[35.6,32.8]-[35.95,33.35]
   - Nagorno-Karabakh (z7+): ~[46.3,39.0]-[47.2,40.0]
   - Khuzestan (z7+): ~[47.5,30.0]-[50.5,33.0]
   Each feature has properties: `{ name: string; minzoom: number }`

**politicalPatterns.ts** -- Create canvas pattern generation module:

1. `PATTERN_SIZE = 16` constant (power of 2 for seamless tiling)

2. `createHatchPattern(color: [number, number, number], alpha: number, lineWidth?: number, spacing?: number): ImageData` -- generates 45-degree diagonal line pattern on transparent background. Default lineWidth=2, spacing=6. Draw diagonal lines that tile seamlessly across the 16x16 canvas.

3. `createDualHatchPattern(color1: [number, number, number], color2: [number, number, number], alpha: number): ImageData` -- alternating diagonal stripes in two faction colors for contested countries. Alternates color1 and color2 stripes with 4px offset between them.

4. `registerPatterns(map: maplibregl.Map): void` -- registers all pattern images:
   - `'hatch-iran'`: createHatchPattern([239,68,68], 0.35)
   - `'hatch-us'`: createHatchPattern([59,130,246], 0.35)
   - `'hatch-turkic'`: createHatchPattern([245,158,11], 0.35)
   - `'hatch-neutral'`: createHatchPattern([156,163,175], 0.2) (lower opacity for neutral)
   - `'hatch-contested_iran_us'`: createDualHatchPattern([239,68,68], [59,130,246], 0.35)
   - `'hatch-contested_iran_china_us'`: createDualHatchPattern([239,68,68], [59,130,246], 0.35)
   Uses `map.hasImage(name)` guard before `map.addImage(name, imageData)`.

5. Import maplibre-gl types only (not the library itself -- the map instance is passed in).
