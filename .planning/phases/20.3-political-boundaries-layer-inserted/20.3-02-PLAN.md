**PoliticalOverlay.tsx** -- Create the overlay component following GeographicOverlay.tsx pattern (Source/Layer children of Map, gated by layerStore):

1. Import `useEffect`, `useState` from React; `Source`, `Layer`, `useMap` from `@vis.gl/react-maplibre`; `useLayerStore` from layerStore; `BORDER_LAYERS` from constants; `registerPatterns` from politicalPatterns; `FACTIONS`, `COUNTRIES_GEOJSON`, `DISPUTED_GEOJSON` from politicalData.

2. Gate on `useLayerStore((s) => s.activeLayers.has('political'))`.

3. Pattern registration with timing guard:
   - Track `patternsReady` state via `useState(false)`.
   - In a `useEffect` keyed on `[map, isActive]`: when map exists and layer is active, call `registerPatterns(map)` then `setPatternsReady(true)`. When layer becomes inactive, set `setPatternsReady(false)`.
   - CRITICAL: Only render fill layers when `patternsReady` is true (prevents blank fills from rendering before addImage completes).

4. CARTO border toggle:
   - In a separate `useEffect` keyed on `[map, isActive]`: iterate `BORDER_LAYERS`, for each layer ID check `map.getLayer(id)` exists, then `map.setLayoutProperty(id, 'visibility', isActive ? 'none' : 'visible')`.
   - Return cleanup function that restores CARTO borders to `'visible'` (prevents borders staying hidden if component unmounts while active).

5. Early return `null` if `!isActive || !patternsReady`.

6. Render structure:
   ```
   <>
     <Source id="political-countries" type="geojson" data={COUNTRIES_GEOJSON}>
       {/* One fill layer per faction, each with static fill-pattern */}
       {FACTIONS.map(({ id, isoCodes, patternName }) => (
         <Layer
           key={id}
           id={`political-fill-${id}`}
           type="fill"
           filter={['in', ['get', 'ADM0_A3'], ['literal', isoCodes]]}
           paint={{ 'fill-pattern': patternName }}
         />
       ))}
       {/* Faction-colored border lines -- thick 3px */}
       <Layer
         id="political-borders"
         type="line"
         paint={{
           'line-color': /* data-driven match expression mapping ADM0_A3 to faction hex color */,
           'line-width': 3,
           'line-opacity': 0.9,
         }}
       />
     </Source>
     <Source id="political-disputed" type="geojson" data={DISPUTED_GEOJSON}>
       {/* Dashed white outlines for disputed zones */}
       <Layer
         id="political-disputed-outline"
         type="line"
         paint={{
           'line-color': '#ffffff',
           'line-width': 2,
           'line-dasharray': [4, 3],
           'line-opacity': 0.8,
         }}
       />
       {/* Zone name labels -- uppercase, letter-spaced */}
       <Layer
         id="political-disputed-labels"
         type="symbol"
         layout={{
           'text-field': ['get', 'name'],
           'text-size': 10,
           'text-letter-spacing': 0.15,
           'text-transform': 'uppercase',
           'text-font': ['Open Sans Bold', 'Arial Unicode MS Bold'],
         }}
         paint={{
           'text-color': '#ffffff',
           'text-halo-color': '#000000',
           'text-halo-width': 1,
         }}
       />
     </Source>
   </>
   ```

7. For the border line color expression, build a `['match', ['get', 'ADM0_A3'], ...pairs, fallbackColor]` expression. Iterate the FACTIONS array to build the pairs: for each faction, expand each isoCode to `[isoCode, faction.color]`. Use `'#666666'` as fallback for unassigned countries.

8. For disputed zone labels, use per-feature `minzoom` from DISPUTED_GEOJSON properties. Since MapLibre `minzoom` on a Layer applies to ALL features, and we need per-feature zoom thresholds, use a filter expression: `['<=', ['get', 'minzoom'], ['zoom']]` -- but note that `['zoom']` is NOT valid inside filter expressions in MapLibre. Instead, create TWO symbol layers:
   - `political-disputed-labels-z5` with `minzoom={5}` and filter `['<=', ['get', 'minzoom'], 5]` (shows z5+ features at z5+)
   - `political-disputed-labels-z7` with `minzoom={7}` and filter `['==', ['get', 'minzoom'], 7]` (shows z7+ features at z7+)
   Alternatively, use a single symbol layer with `minzoom={5}` and a `['step', ['zoom'], false, 5, ['<=', ['get', 'minzoom'], 5], 7, true]` visibility expression in `layout.visibility` -- but `visibility` only accepts `'visible'`/`'none'`, not expressions. The simplest correct approach: use `'icon-opacity'` and `'text-opacity'` with a step expression on zoom, OR split into two layers. Prefer two layers for clarity.
