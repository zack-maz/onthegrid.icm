# Phase 20.3: Political Boundaries Layer - Research

**Researched:** 2026-03-23
**Domain:** MapLibre GL JS fill-pattern hatching, Natural Earth GeoJSON, visualization layer integration
**Confidence:** HIGH

## Summary

This phase adds a toggleable political boundaries overlay that color-codes countries by faction alignment using diagonal hatching fills, renders thick faction-colored borders, and marks disputed territories with dashed outlines. The core technical challenge is implementing per-faction diagonal hatching patterns via MapLibre's `fill-pattern` paint property, which does NOT support data-driven styling -- meaning we need multiple fill layers (one per faction), each with its own pre-registered canvas-generated pattern image.

The data source is Natural Earth 110m countries GeoJSON (~500KB raw, ~50KB gzipped), imported as a static asset following the same pattern as `geoFeatures.ts`. Disputed territory boundaries are NOT available at 110m scale in Natural Earth, so they must be hand-drawn as simple GeoJSON polygons. The layer integrates cleanly with the existing layerStore (`'political'` already defined), LayerTogglesSlot (already has a 'Political' row with `comingSoon: true`), and LEGEND_REGISTRY (needs extension for discrete swatches instead of gradient bars).

**Primary recommendation:** Use canvas-generated ImageData patterns registered via imperative `map.addImage()` in a useEffect hook, with one fill layer per faction filtered by ISO_A3 property, plus a separate line layer for borders and a dashed-line layer for disputed territories.

### Pattern 1: Canvas-Generated Fill Patterns via addImage
**What:** Generate diagonal hatching pattern tiles as ImageData using the Canvas API, register them with `map.addImage()` imperatively, then reference by name in `fill-pattern` paint property.
**When to use:** When fill-pattern does not support data-driven styling (current MapLibre limitation).
**Why this approach:**
- `fill-pattern` does NOT support data-driven expressions in MapLibre GL JS (confirmed via official style spec docs)
- This means we cannot use a single fill layer with `['match', ['get', 'faction'], ...]` for patterns
- Instead: register one named pattern image per faction, create one fill layer per faction with a `filter` expression

**Example:**
```typescript
// politicalPatterns.ts
const PATTERN_SIZE = 16; // Must be power of 2 for seamless tiling

export function createHatchPattern(
  color: [number, number, number],
  alpha: number,
  lineWidth: number = 2,
  spacing: number = 6,
): ImageData {
  const canvas = document.createElement('canvas');
  canvas.width = PATTERN_SIZE;
  canvas.height = PATTERN_SIZE;
  const ctx = canvas.getContext('2d')!;

  // Transparent background
  ctx.clearRect(0, 0, PATTERN_SIZE, PATTERN_SIZE);

  // Draw 45-degree diagonal lines
  ctx.strokeStyle = `rgba(${color[0]}, ${color[1]}, ${color[2]}, ${alpha})`;
  ctx.lineWidth = lineWidth;

  // Draw diagonal lines that tile seamlessly
  for (let i = -PATTERN_SIZE; i < PATTERN_SIZE * 2; i += spacing) {
    ctx.beginPath();
    ctx.moveTo(i, 0);
    ctx.lineTo(i + PATTERN_SIZE, PATTERN_SIZE);
    ctx.stroke();
  }

  return ctx.getImageData(0, 0, PATTERN_SIZE, PATTERN_SIZE);
}

// Registration in component via useMap + useEffect
export function registerPatterns(map: maplibregl.Map): void {
  const patterns = {
    'hatch-iran': createHatchPattern([239, 68, 68], 0.35),
    'hatch-us': createHatchPattern([59, 130, 246], 0.35),
    'hatch-turkic': createHatchPattern([245, 158, 11], 0.35),
    'hatch-neutral': createHatchPattern([156, 163, 175], 0.2),
  };
  for (const [name, imageData] of Object.entries(patterns)) {
    if (!map.hasImage(name)) {
      map.addImage(name, imageData);
    }
  }
}
```

### Pattern 2: Multiple Fill Layers with Filter Expressions
**What:** Since fill-pattern is not data-driven, use one `<Layer type="fill">` per faction, each with a `filter` expression matching ISO_A3 codes and a fixed `fill-pattern` referencing the faction's registered pattern.
**When to use:** Always for this phase -- it's the only reliable approach.

**Example:**
```typescript
// PoliticalOverlay.tsx

  turkic: ['TUR', 'AZE', 'TKM'],
  neutral: ['OMN', 'AFG', /* ... remaining */],

// For each faction:
<Layer

  source="political-countries"
  filter={['in', ['get', 'ISO_A3'], ['literal', isoCodes]]}

```

### Pattern 3: Contested Country Dual-Stripe Patterns
**What:** Generate special alternating-stripe patterns for contested countries by drawing two colors in alternating diagonal bands.
**When to use:** Iraq, Lebanon, Pakistan (countries with split faction loyalty).

**Example:**
```typescript
export function createDualHatchPattern(
  color1: [number, number, number],
  color2: [number, number, number],
  alpha: number,
): ImageData {
  const size = 16;
  const canvas = document.createElement('canvas');
  canvas.width = size;
  canvas.height = size;
  const ctx = canvas.getContext('2d')!;
  ctx.clearRect(0, 0, size, size);

  // Alternating diagonal stripes in two colors
  for (let i = -size; i < size * 2; i += 8) {
    // Color 1 stripe
    ctx.strokeStyle = `rgba(${color1[0]}, ${color1[1]}, ${color1[2]}, ${alpha})`;
    ctx.lineWidth = 2;
    ctx.beginPath();
    ctx.moveTo(i, 0);
    ctx.lineTo(i + size, size);
    ctx.stroke();

    // Color 2 stripe (offset)
    ctx.strokeStyle = `rgba(${color2[0]}, ${color2[1]}, ${color2[2]}, ${alpha})`;
    ctx.beginPath();
    ctx.moveTo(i + 4, 0);
    ctx.lineTo(i + 4 + size, size);
    ctx.stroke();
  }

  return ctx.getImageData(0, 0, size, size);
}
```

### Pattern 5: Discrete-Swatch Legend (New Legend Variant)
**What:** The existing LEGEND_REGISTRY/LegendItem pattern renders gradient bars. Political layer needs discrete colored dots/swatches with faction names.
**Approach:** Extend the `LegendConfig` interface with an optional `mode: 'gradient' | 'discrete'` field. When mode is 'discrete', render swatches instead of a gradient bar. This is backward-compatible.

**Example:**
```typescript
// MapLegend.tsx extension
export interface LegendConfig {
  layerId: VisualizationLayerId;
  title: string;
  colorStops: Array<{ color: string; label: string }>;
  mode?: 'gradient' | 'discrete'; // NEW: defaults to 'gradient'
}

// New rendering path in LegendItem:
if (config.mode === 'discrete') {
  return (
    <div className="pointer-events-auto rounded bg-surface-overlay/80 px-2 py-1.5 backdrop-blur-sm">
      <div className="mb-1 text-[9px] font-semibold uppercase tracking-wider text-text-secondary">
        {config.title}
      </div>
      <div className="flex flex-col gap-0.5">
        {config.colorStops.map((stop) => (
          <div key={stop.label} className="flex items-center gap-1.5">
            <span className="inline-block h-2 w-2 rounded-full" style={{ backgroundColor: stop.color }} />
            <span className="text-[8px] text-text-muted">{stop.label}</span>
          </div>
        ))}
      </div>
    </div>
  );
}
```

### Anti-Patterns to Avoid
- **Single fill layer with data-driven fill-pattern:** MapLibre fill-pattern does NOT support data-driven expressions. This will silently fail or show no pattern at all. Use multiple layers with filters instead.
- **Loading external pattern images via URL:** Unnecessary network requests; canvas-generated patterns are faster and have zero latency.
- **Storing GeoJSON in a server route:** Natural Earth 110m is static data (~200KB). Import it directly as a client-side asset, same as geoFeatures.ts.
- **Using feature-state for pattern switching:** Confirmed broken in MapLibre issue #4930. Use feature properties + filter instead.
- **Forgetting power-of-2 pattern dimensions:** MapLibre requires pattern image width/height to be factors of 2 (2, 4, 8, 16, 32, etc.) for seamless tiling.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Country boundary polygons | Custom GeoJSON from scratch | Natural Earth 110m ne_110m_admin_0_countries | 258 countries, ISO codes, professionally maintained |
| Diagonal line pattern generation | Complex SVG pattern system | Canvas API + ImageData | 20 lines of canvas code per pattern, no dependencies |
| ISO country code lookups | Manual code-to-name mapping | Natural Earth ISO_A3 property | Already in the GeoJSON features |

**Key insight:** The disputed territory zones (Kurdistan, Gaza, etc.) are the only hand-drawn data. Country boundaries come from Natural Earth. The hatching patterns are generated programmatically, not sourced as images.

## Common Pitfalls

### Pitfall 1: fill-pattern Not Data-Driven
**What goes wrong:** Attempting to use `['match', ['get', 'faction'], 'iran', 'hatch-iran', ...]` as fill-pattern value.
**Why it happens:** MapLibre style spec explicitly states fill-pattern does not support data-driven expressions.
**How to avoid:** Use one fill layer per faction, each with a static `fill-pattern` value and a `filter` expression to select the right countries.
**Warning signs:** Polygons render with no fill at all, or all polygons use the same pattern.

### Pitfall 2: Pattern Image Registration Timing
**What goes wrong:** Layer renders before pattern images are registered, causing blank fills.
**Why it happens:** React declarative rendering can create a Layer before the imperative `addImage()` call completes.
**How to avoid:** Track pattern registration state in a `useState` boolean. Only render fill layers after patterns are registered. Use `map.hasImage()` as a guard.
**Warning signs:** Patterns appear after a second toggle off/on cycle, or only on hot reload.

### Pitfall 3: Natural Earth ISO_A3 Missing Values
**What goes wrong:** Some countries have `-99` as their ISO_A3 value in Natural Earth data.
**Why it happens:** Natural Earth uses `-99` for entities without standard ISO codes (e.g., disputed territories, some dependencies).
**How to avoid:** Use `ADM0_A3` (always populated) as the primary lookup key instead of `ISO_A3`, or handle `-99` as a fallback.
**Warning signs:** Some countries render without any faction hatching despite being in the faction definition.

### Pitfall 4: CARTO Border Toggle Not Restored on Unmount
**What goes wrong:** If the user navigates away or the component unmounts while political layer is active, CARTO borders stay hidden.
**Why it happens:** Missing cleanup function in useEffect.
**How to avoid:** Return a cleanup function that restores CARTO borders to 'visible'.
**Warning signs:** Toggling political layer off leaves a border-less map.

### Pitfall 5: GeoJSON Bundle Size
**What goes wrong:** Importing the full Natural Earth 110m GeoJSON (~500KB raw) bloats the client bundle.
**Why it happens:** The file includes all 258 countries worldwide with dozens of properties per feature.
**How to avoid:** Pre-filter the GeoJSON to only include countries within the map's MAX_BOUNDS (west:20, south:0, east:80, north:50). Strip unnecessary properties, keeping only ISO_A3/ADM0_A3 and geometry. This should reduce to ~80-100KB raw, ~20-30KB gzipped.
**Warning signs:** Lighthouse reports large unused JS; initial load time increases noticeably.

### Pitfall 6: Canvas Not Available in jsdom Tests
**What goes wrong:** `document.createElement('canvas')` returns an element without `getContext('2d')` in jsdom.
**Why it happens:** jsdom does not implement Canvas API.
**How to avoid:** Mock the pattern generation function in tests, or use the vitest mock system to stub `createHatchPattern`.
**Warning signs:** Tests crash with "getContext is not a function" or return null.

### PoliticalOverlay Component Structure
```typescript
// PoliticalOverlay.tsx
import { useEffect, useState } from 'react';
import { Source, Layer, useMap } from '@vis.gl/react-maplibre';
import { useLayerStore } from '@/stores/layerStore';
import { BORDER_LAYERS } from '@/components/map/constants';
import { LEGEND_REGISTRY } from '@/components/map/MapLegend';
import { registerPatterns } from './politicalPatterns';
import { COUNTRIES_GEOJSON, DISPUTED_GEOJSON, FACTIONS } from './politicalData';

export function PoliticalOverlay() {
  const isActive = useLayerStore((s) => s.activeLayers.has('political'));
  const { current: map } = useMap();
  const [patternsReady, setPatternsReady] = useState(false);

  // Register hatching patterns imperatively
  useEffect(() => {
    if (!map || !isActive) {
      setPatternsReady(false);
      return;
    }
    registerPatterns(map);
    setPatternsReady(true);
  }, [map, isActive]);

  // Toggle CARTO basemap borders
  useEffect(() => {
    if (!map) return;
    BORDER_LAYERS.forEach((id) => {
      if (map.getLayer(id)) {
        map.setLayoutProperty(id, 'visibility', isActive ? 'none' : 'visible');
      }
    });
    return () => {
      // Restore on unmount
      BORDER_LAYERS.forEach((id) => {
        if (map.getLayer(id)) {
          map.setLayoutProperty(id, 'visibility', 'visible');
        }
      });
    };
  }, [map, isActive]);

  if (!isActive || !patternsReady) return null;

  return (
    <>
      <Source id="political-countries" type="geojson" data={COUNTRIES_GEOJSON}>
        {FACTIONS.map(({ id, isoCodes, patternName }) => (
          <Layer
            key={id}
            id={`political-fill-${id}`}
            type="fill"
            filter={['in', ['get', 'ADM0_A3'], ['literal', isoCodes]]}
            paint={{ 'fill-pattern': patternName }}
          />
        ))}
        {/* Faction-colored border lines */}
        <Layer
          id="political-borders"
          type="line"
          paint={{
            'line-color': ['match', ['get', 'ADM0_A3'],
              .../* flatten faction-to-color mappings */,
              '#666666', // fallback
            ],
            'line-width': 3,
            'line-opacity': 0.9,
          }}
        />
      </Source>
      <Source id="political-disputed" type="geojson" data={DISPUTED_GEOJSON}>
        <Layer
          id="political-disputed-outline"
          type="line"
          paint={{
            'line-color': '#ffffff',
            'line-width': 2,
            'line-dasharray': [4, 3],
            'line-opacity': 0.8,
          }}
        />
        <Layer
          id="political-disputed-labels"
          type="symbol"
          layout={{
            'text-field': ['get', 'name'],
            'text-size': 10,
            'text-letter-spacing': 0.15,
            'text-transform': 'uppercase',
          }}
          paint={{
            'text-color': '#ffffff',
            'text-halo-color': '#000000',
            'text-halo-width': 1,
          }}
          minzoom={5}
        />
      </Source>
    </>
  );
}

// Legend registration (module scope)
LEGEND_REGISTRY.push({
  layerId: 'political',
  title: 'Factions',
  mode: 'discrete',
  colorStops: [
    { color: '#ef4444', label: 'Iran Axis' },
    { color: '#3b82f6', label: 'US-Aligned' },
    { color: '#f59e0b', label: 'Turkic Bloc' },
    { color: '#9ca3af', label: 'Contested' },
    { color: '#6b7280', label: 'Neutral' },
  ],
});
```

## Open Questions

1. **Natural Earth GeoJSON: ISO_A3 vs ADM0_A3 reliability**
   - What we know: Some countries have `-99` as ISO_A3; ADM0_A3 is always populated
   - What's unclear: Exact list of affected countries in the Middle East region
   - Recommendation: Use ADM0_A3 as the primary lookup key; verify during implementation that all target countries match

2. **Contested country pattern tiling quality**
   - What we know: Dual-stripe patterns with alternating colors are possible via canvas
   - What's unclear: Whether the visual result looks clean at all zoom levels with 16px tile size
   - Recommendation: Start with 16px pattern, increase to 32px if stripe alternation looks choppy

3. **Egypt faction placement**
   - What we know: Egypt receives significant US military aid but maintains diplomatic ties with Iran
   - What's unclear: User preference for current conflict context
   - Recommendation: Place Egypt as US-Aligned (consistent with Camp David Accords, US military aid recipient, normalized relations with Israel)
