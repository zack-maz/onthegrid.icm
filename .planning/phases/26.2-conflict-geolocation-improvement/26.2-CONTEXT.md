# Phase 26.2: Conflict Geolocation Improvement - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Reduce misplaced conflict events on the map by adding NLP-based cross-validation to the GDELT pipeline. Extract actors and locations from article titles using compromise (free, local, already installed), cross-check against GDELT's geocoded coordinates, reject mismatches, and relocate centroid events to specific cities when NLP identifies a place name. No paid API fees — all NLP runs locally in the serverless function.

</domain>

<decisions>
## Implementation Decisions

### Actor-Location Cross-Validation
- Reject events where NLP-extracted actors clearly contradict the geocoded country (e.g., "Israel strikes Hamas" geocoded to Baghdad → reject)
- Use BOTH GDELT's Actor1/Actor2 fields AND NLP-extracted actors from the article title — more signal = fewer false rejections
- Apply cross-validation to ALL events regardless of confidence — high-confidence misplacements are the most dangerous
- Events with NO actor data (empty Actor1/Actor2) pass through — existing confidence scoring already penalizes sparse metadata
- Bundle actor and location extraction in one NLP pass — compromise parses the title once for both checks
- Rejected events get `rejected:actor_geo_mismatch` in pipeline trace metadata (visible in audit script)

### Location Disambiguation
- NLP-relocate centroid events when possible — extract city/region names from article title, look up coordinates, reassign
- Falls back to centroid + dispersion if NLP can't extract a specific place name
- Expanded CITY_CENTROIDS list generated from GeoNames free dump (~150-200 ME cities, filtered from pop >50k in bbox)
- Ambiguous place names (same name in multiple countries) resolved by picking the city nearest to GDELT's original geocode
- Remove centroid confidence penalty for successfully relocated events — they now have real coordinates

### Free NLP Tooling
- Stick with compromise (already installed, ~200KB, runs in Vercel serverless)
- Add a custom lexicon plugin from the GeoNames city list so compromise.places() recognizes ME-specific city names (Isfahan, Mosul, Homs, Deir ez-Zor, etc.)
- NLP runs server-side in the parseAndFilter pipeline on article titles fetched from SOURCEURL

### Title Fetching Strategy
- Fetch article title from SOURCEURL via HTTP GET of HTML head, parse og:title or <title> tag (~5-10KB per request)
- Batch title fetches with concurrency limit (e.g., 10 at once) to avoid overwhelming news sites
- Cache title extractions in Redis (URL → title, 7-day TTL) — most URLs repeat across GDELT updates
- Title fetch failures → skip NLP enrichment but penalize confidence (0.7x multiplier) since event couldn't be independently verified
- Title fetched + NLP mismatch → reject event
- Title fetched + NLP confirms → full confidence

### Filter Threshold Tuning
- Primary problem is mismatched events (wrong location), not over/under-filtering — NLP cross-validation is the primary fix
- Tighten confidence threshold slightly (exact value is Claude's discretion)
- Reject low-tier CAMEO codes entirely (180, 182, 190) instead of scoring at 0.1x — these vague codes produce the most noise
- Remove centroid penalty for NLP-relocated events; keep penalty for events that couldn't be relocated

### Claude's Discretion
- Actor-country matching strategy (lookup table vs broad matching — informed by actual GDELT data patterns)
- Exact confidence threshold values after NLP layer is active
- Concurrency limit for title fetching (5-15 range)
- GeoNames population threshold and filtering criteria for city list generation
- Redis cache key format for title extractions
- How to handle GDELT events with multiple SOURCEURLs

</decisions>

## Specific Ideas

- "I don't want to solely rely on GDELT" — NLP should be authoritative when title is fetchable, not just advisory
- GeoNames dump should be filtered slightly (not raw) — population threshold + ME bbox
- The core issue is mismatched events (wrong location on the map), not too many or too few events overall
- Zero paid APIs — everything runs locally in the serverless function via compromise
