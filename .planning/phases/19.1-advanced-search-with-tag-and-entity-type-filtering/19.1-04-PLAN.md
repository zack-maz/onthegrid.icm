    **Create `src/components/search/TagChipRow.tsx`:**

    A horizontal row of clickable chips displayed above the search input.

    1. Default chips (always shown, in this order):
       - [Events] -> inserts `type:airstrike OR type:ground_combat OR type:shelling OR type:bombing OR type:assassination OR type:abduction`
       - [Sites] -> inserts `type:site`
       - [Iran] -> inserts `country:iran`
       - [US] -> inserts `country:united states`

    2. Recent tag chips: read `recentTags` from searchStore, render after defaults
       - Each recent tag is a single tag expression (e.g., `country:iraq`, `type:flight`)
       - Deduplicate against default chip values

    3. Chip styling: small rounded pills with bg-surface border-border text-text-muted, hover:bg-surface-elevated
       - No dots or bullet indicators (per locked decision)

    4. onClick: append the chip's tag expression to the current query string (with a space separator if query is non-empty), then focus the input

    Props: `onInsertTag: (tag: string) => void`

    **Create `src/components/search/SyntaxOverlay.tsx`:**

    A div that mirrors the input text with colored spans for tag highlighting.

    1. Use Pattern 2 from research: transparent input on top, colored overlay behind
    2. Call `tokenize(query)` to get tokens with positions
    3. For each token:
       - TAG tokens: render with color from `getTagColor(token.prefix)`. Show prefix in color, `:` in dim, value in slightly lighter shade
       - AND/OR tokens: render in text-text-muted italic
       - TEXT tokens: render in text-text-primary (default)
       - Parentheses: render in text-text-muted
       - Invalid/unknown prefixes: render with text-red-400 (red tint)
    4. Match font, padding, letter-spacing exactly with the input element
    5. `pointer-events-none` and `aria-hidden` on overlay div

    Props: `query: string, className?: string`

    **Create `src/hooks/useAutocomplete.ts`:**

    Computes autocomplete suggestions from current cursor position in the query string.

    1. Two-stage logic:
       - Stage 1 (prefix suggestions): When user is typing a word that partially matches a tag prefix.
         - Detect the word under cursor (between whitespace boundaries)
         - If it partially matches a prefix (e.g., "co" matches "country:", "callsign:"), suggest matching prefixes
         - Each suggestion: `{ type: 'prefix', prefix: string, label: string, description: string }`
       - Stage 2 (value suggestions): When user has typed a complete prefix followed by `:` and partial value.
         - Detect `prefix:partialValue` pattern at cursor
         - Call `getTagValues(prefix, entityDataSources)` to get known values
         - Filter by partial value match
         - Each suggestion: `{ type: 'value', prefix: string, value: string, count: number }`

    2. Use `useMemo` with entity arrays as deps to pre-compute available values once per data update (not per keystroke, per research Pitfall 3)
    3. Debounce autocomplete computation: 100ms delay after last keystroke

    Export: `useAutocomplete(query: string, cursorPosition: number): AutocompleteSuggestion[]`

    **Create `src/components/search/AutocompleteDropdown.tsx`:**

    Dropdown that renders below the input showing autocomplete suggestions.

    1. Receives suggestions array and selected index from parent
    2. Each suggestion row shows:
       - Stage 1 (prefix): prefix name + description + "Tab to complete"
       - Stage 2 (value): value + count badge (e.g., "iran (24)")
    3. Highlight selected row with bg-surface-elevated
    4. onClick on a row -> accept that suggestion
    5. Styling: absolute positioned below input, bg-surface-elevated border rounded-lg shadow-lg, max-h-[200px] overflow-y-auto
    6. Empty state: nothing rendered (no dropdown when no suggestions)

    Props: `suggestions: AutocompleteSuggestion[], selectedIndex: number, onSelect: (suggestion) => void`
