  <name>Task 1: Create Vercel entry point + config + test</name>
  <files>api/index.ts, vercel.json, package.json, tsconfig.server.json, server/__tests__/vercel-entry.test.ts</files>
  <behavior>
    - vercel-entry.test: api/index.ts module has a default export
    - vercel-entry.test: default export is a function (Express app)
    - vercel-entry.test: app responds to GET /health with 200
  </behavior>
  <action>
    1. Create api/index.ts — the Vercel serverless entry point:
       ```typescript
       import { createApp } from '../server/index.js';

       const app = createApp();
       export default app;
       ```
       This is intentionally minimal. Vercel's Express integration detects `export default app` and wraps it as a serverless function. No app.listen() here.

    2. Create vercel.json — deployment configuration per user decisions:
       ```json
       {
         "$schema": "https://openapi.vercel.sh/vercel.json",
         "framework": "vite",
         "functions": {
           "api/index.ts": {
             "maxDuration": 60
           }
         },
         "rewrites": [
           { "source": "/api/:path*", "destination": "/api" },
           { "source": "/(.*)", "destination": "/index.html" }
         ]
       }
       ```
       Rewrite order: API routes first (catch-all function), then SPA fallback. Framework: "vite" for frontend auto-detection. maxDuration: 60s per user decision (AISStream WebSocket collect needs headroom).

    3. Update package.json — add `"engines": { "node": "22.x" }` field to pin Node 22.x per user decision. Do NOT change any scripts or dependencies.

    4. Update tsconfig.server.json — add "api" to the include array so TypeScript type-checks api/index.ts:
       ```json
       "include": ["server", "api"]
       ```

    5. Create server/__tests__/vercel-entry.test.ts — unit test for the entry point:
       - Use `@vitest-environment node` inline directive
       - Mock server dependencies (config, redis, adapters) same as server.test.ts
       - Import api/index.ts default export
       - Test: default export exists and is a function
       - Test: GET /health returns 200 with { status: 'ok' }
       - Use supertest-style fetch against the app (same pattern as server.test.ts: app.listen(0) on random port, fetch, close)

    6. Run full test suite: `npx vitest run` to verify no regressions across frontend and server.

    7. Run `npx tsc -b` to verify TypeScript compiles cleanly with the new api/ directory included.
  </action>
  <verify>
    <automated>npx vitest run server/__tests__/vercel-entry.test.ts -x && npx tsc -b --noEmit</automated>
  </verify>
  <done>api/index.ts exports Express app as default; vercel.json has rewrites and function config; package.json pins Node 22.x; tsconfig includes api/; entry point test passes; tsc -b clean</done>
</task>
