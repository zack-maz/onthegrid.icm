1. **Create `src/stores/flightStore.ts`:**
   Follow the curried `create<T>()()` pattern from mapStore. Define:
   ```
   type ConnectionStatus = 'connected' | 'stale' | 'error' | 'loading';

   interface FlightState {
     flights: FlightEntity[];
     connectionStatus: ConnectionStatus;
     lastFetchAt: number | null;
     lastFresh: number | null;
     flightCount: number;
     setFlightData: (response: CacheResponse<FlightEntity[]>) => void;
     setError: () => void;
     setLoading: () => void;
     clearStaleData: () => void;
   }
   ```
   - `setFlightData`: Full atomic replace of flights array. Set `flightCount: response.data.length`. Set `connectionStatus: response.stale ? 'stale' : 'connected'`. Set `lastFetchAt: Date.now()`. For `lastFresh`: if `response.stale` is false, set `Date.now()`; if `response.stale` is true, keep the existing `lastFresh` value (use `get()` to read current state).
   - `setError`: Set `connectionStatus: 'error'`.
   - `setLoading`: Set `connectionStatus: 'loading'`.
   - `clearStaleData`: Set `flights: [], flightCount: 0, connectionStatus: 'error'`.
   - Import `FlightEntity` and `CacheResponse` from `@/types/entities`.

2. **Create `src/hooks/useFlightPolling.ts`:**
   Export a `useFlightPolling()` hook. Constants: `POLL_INTERVAL = 5_000`, `STALE_THRESHOLD = 60_000` (60 seconds -- flights traveling at 250m/s drift ~15km in 60s, making positions meaningfully outdated for intelligence use).

   Implementation:
   - Use `useRef<ReturnType<typeof setTimeout> | null>(null)` for the timeout ID.
   - Use Zustand selectors to get `setFlightData`, `setError`, `setLoading`, `clearStaleData`, `lastFresh` from `useFlightStore`.
   - `fetchFlights` async function: fetch `/api/flights`, parse JSON as `CacheResponse<FlightEntity[]>`, call `setFlightData(data)`. On error, call `setError()`.
   - `schedulePoll` function: sets `timeoutRef.current = setTimeout(async () => { await fetchFlights(); checkStaleness(); schedulePoll(); }, POLL_INTERVAL)`.
   - `checkStaleness` function: reads `useFlightStore.getState().lastFresh`. If `lastFresh` exists and `Date.now() - lastFresh > STALE_THRESHOLD`, call `clearStaleData()`.
   - `useEffect` on mount: call `setLoading()`, then `fetchFlights().then(schedulePoll)`.
   - Visibility change handler inside the effect: on `'hidden'` clear the timeout. On `'visible'` call `fetchFlights().then(schedulePoll)` (immediate fetch then resume).
   - Cleanup: clear timeout, remove visibilitychange listener.
   - Use `useCallback` for `fetchFlights` and `schedulePoll` with proper dependencies, OR use the ref-based approach to avoid stale closures (preferred: store setters from Zustand are stable references, so useCallback deps are minimal).

   **Important anti-patterns to avoid:**
   - Do NOT use `setInterval` (does not wait for async completion).
   - Do NOT store the fetch function in Zustand (keep polling orchestration in the hook).
   - Do NOT use AbortController (5s timeout is already short, adds complexity without benefit).

3. **Create `src/__tests__/flightStore.test.ts`:**
   Test the store in isolation (no React rendering needed -- Zustand stores are testable with `getState()/setState()`):
   - Reset store state in `beforeEach` using `useFlightStore.setState(...)`.
   - Test initial state matches expected defaults.
   - Test `setFlightData` with `{ data: [mockFlight], stale: false, lastFresh: Date.now() }` sets connected + correct count.
   - Test `setFlightData` with `stale: true` sets 'stale' status and preserves existing lastFresh.
   - Test `setError` sets 'error' status.
   - Test `setLoading` sets 'loading' status.
   - Test `clearStaleData` empties flights and sets error.
   - Create a minimal mock FlightEntity for use in tests.

4. **Create `src/__tests__/useFlightPolling.test.ts`:**
   Use `vi.useFakeTimers()` and `vi.stubGlobal('fetch', mockFetch)`:
   - Test: on mount, fetch is called once immediately (before any timer fires).
   - Test: after advancing timers by 5000ms, fetch is called again.
   - Test: dispatching a `visibilitychange` event with `document.visibilityState = 'hidden'` clears the scheduled timeout (advance 10s, no extra fetch calls).
   - Test: dispatching `visibilitychange` back to 'visible' triggers an immediate fetch.
   - Test: on unmount (cleanup), the timeout is cleared.
   - Use `renderHook` from `@testing-library/react` to mount/unmount the hook.
   - Mock `document.visibilityState` as a getter using `vi.spyOn(document, 'visibilityState', 'get')`.
   - Mock fetch to return a valid CacheResponse: `{ data: [], stale: false, lastFresh: Date.now() }`.
