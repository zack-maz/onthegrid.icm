**1. Create `src/stores/searchStore.ts`:**
```typescript
interface SearchState {
  query: string;
  isSearchModalOpen: boolean;
  isFilterMode: boolean;
  matchedIds: Set<string>;

  setQuery: (query: string) => void;
  openSearchModal: () => void;
  closeSearchModal: () => void;
  applyAsFilter: () => void;
  clearSearch: () => void;
  setMatchedIds: (ids: Set<string>) => void;
}
```
Use curried `create<SearchState>()((set) => ({...}))` pattern per project convention.
- `openSearchModal`: `{ isSearchModalOpen: true }`
- `closeSearchModal`: `{ isSearchModalOpen: false }` (do NOT clear query -- user might want to re-open)
- `applyAsFilter`: `{ isFilterMode: true, isSearchModalOpen: false }` (close modal, keep query)
- `clearSearch`: `{ query: '', isFilterMode: false, matchedIds: new Set() }`

**2. Create `src/lib/searchUtils.ts`:**
Pure functions, no React/store dependencies.
- `getSearchableFields(entity: MapEntity | SiteEntity): string[]` -- returns lowercase string values for all searchable fields per entity type (see interfaces above). For flight: [label, icao24, originCountry]. For ship: [label, mmsi, shipName]. For event: [label, type, actor1, actor2, locationName]. For site: [label, siteType, operator]. Filter out null/undefined values.
- `searchEntities<T extends { id: string }>(query: string, entities: T[], fieldExtractor: (e: T) => string[]): SearchResult<T>[]` -- returns entities where any field contains the query substring (case-insensitive). Each result includes the entity, the matched field name, and the matched value. Return empty array if query is empty/whitespace.
- `export interface SearchResult<T> { entity: T; matchField: string; matchValue: string; }`

**3. Create `src/hooks/useSearchResults.ts`:**
Hook that reads from all 4 entity stores and returns grouped search results.
- Read query from searchStore
- Read flights from flightStore, ships from shipStore, events from eventStore (use useFilteredEntities for filtered data), sites from siteStore
- Use useMemo with [query] as dependency (NOT entity arrays -- per research Pitfall 2). Use useRef to hold latest entity data so results don't recompute on every poll.
- Call searchUtils.searchEntities for each entity type
- Cap results at 10 per entity type (40 total max)
- Return: `{ flights: SearchResult[], ships: SearchResult[], events: SearchResult[], sites: SearchResult[], totalCount: number }`

**4. Extend `src/stores/filterStore.ts`:**
In the `clearAll` action, after existing reset logic, also call `useSearchStore.getState().clearSearch()`. Import useSearchStore at the top of filterStore.
