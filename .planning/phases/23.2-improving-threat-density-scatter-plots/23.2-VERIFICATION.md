### Human Verification Required

The following behaviors pass automated checks but require visual confirmation:

#### 1. Radial Gradient Appearance

**Test:** Enable Threat layer on the map with events loaded. Zoom to a region with multiple clusters.
**Expected:** Cluster circles glow from a bright center, fading to transparent at the edge. No hard circle edges visible. Where circles overlap, the intersection is visibly brighter than the individual circles.
**Why human:** GLSL shader output, additive blending, and canvas compositing cannot be verified without a GPU/rendering context in the test environment.

#### 2. Zoom Z-Order Crossover

**Test:** Zoom the map smoothly from zoom level 7 to zoom level 11 while viewing an area with both clusters and event markers.
**Expected:** At zoom < 9, cluster circles appear in front of event markers. At zoom >= 9, event markers appear on top of cluster circles. The crossover should be instantaneous (not gradual) as zoom crosses 9.
**Why human:** Deck.gl layer ordering affects GPU draw order — only visible in a live rendering context.

#### 3. Hover Dimming

**Test:** With Threat layer active and multiple clusters visible, hover the cursor over one cluster.
**Expected:** The hovered cluster brightens (full opacity). All other clusters dim to approximately 40% opacity. Moving cursor away restores all clusters to default opacity.
**Why human:** Alpha modulation requires live rendering and user interaction to verify.

#### 4. Geographic Context in Detail Panel

**Test:** Click a threat cluster near a known site (e.g., near an airbase or nuclear plant). Then click a cluster in a remote area with no known sites.
**Expected:** First cluster shows "Near [Site Name]" with a pin icon. Second cluster shows a loading placeholder briefly, then a city/country name (e.g., "Baghdad, Iraq") fetched from the geocode API.
**Why human:** Two-tier geocoding behavior involves async API calls and site proximity matching that require live data.
