    - setLoading: status='loading'
    - No clearStaleData action (events are historical, never cleared for staleness)

    useShipPolling:
    - On mount: calls setLoading, fetches /api/ships, starts 30s recursive setTimeout
    - On successful fetch: calls setShipData with response
    - On fetch error: calls setError
    - On tab hidden: clears timeout (pauses polling)
    - On tab visible: immediate fetch then resume polling
    - After 120s with no fresh data (lastFresh check): calls clearStaleData
    - On unmount: clears timeout, removes visibilitychange listener

    useEventPolling:
    - On mount: calls setLoading, fetches /api/events, starts 300s recursive setTimeout
    - On successful fetch: calls setEventData with response
    - On fetch error: calls setError
    - On tab hidden: clears timeout (pauses polling)
    - On tab visible: immediate fetch then resume polling
    - NO stale threshold check (events don't go stale)
    - On unmount: clears timeout, removes visibilitychange listener
