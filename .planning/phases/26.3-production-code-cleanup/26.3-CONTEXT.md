<decisions>
## Implementation Decisions

### Dead Code Removal
- Remove ALL Phase 26.2 NLP files: nlpGeoValidator.ts, titleFetcher.ts, nlpExtractor.ts changes (revert to pre-26.2), me-cities.json, extract-geonames.ts
- Remove Phase C wiring from gdelt.ts (revert parseAndFilter to pre-26.2 state)
- Remove associated test files: nlpGeoValidator.test.ts, titleFetcher.test.ts
- Remove Phase 26.2 changes to eventScoring.ts, eventAudit.ts, geoValidation.ts, config.ts
- Keep pre-26.2 code intact (CITY_CENTROIDS at 42 entries, original FIPS tables, original CAMEO exclusions)
- Flag hardcoded tables as technical debt for Phase 26.2 GDELT redo — do NOT remove without replacement

### Error Handling & Observability
- Consistent JSON error shape across ALL API routes: `{ error: string, code: string, statusCode: number }`
- Dev includes stack traces, production strips them (NODE_ENV controlled)
- X-Request-ID header: generate UUID per request, attach to all logs and error responses
- Upgrade from custom log() wrapper to pino structured logger (~50KB, JSON-native, child loggers, request context)

### Test Quality
- Remove all `it.todo()` stubs that won't be implemented
- Every remaining test must assert real behavior (not just "renders without crashing")
- Add vitest coverage reporting with 80% line coverage threshold
- Coverage report visible in CI output

### Config & Magic Numbers
- Consolidate server/constants.ts + server/config.ts into single typed config
- Every value has a typed default and can be overridden by env var
- Config validation on startup: crash on missing required vars (UPSTASH_REDIS_REST_URL), warn on invalid optional vars
- All polling intervals, Redis TTLs, thresholds, bbox coords in one place

### Input Validation & Security
- Zod schemas for all API query params (type check, range check, sanitize strings)
- No raw req.query access — all params go through Zod parse
- Validate all env vars at startup using Zod schema

### Graceful Shutdown
- SIGTERM handler drains in-flight requests, closes Redis connections, exits cleanly
- Required for production serverless (Vercel function lifecycle)

### API Response Compression
- gzip/brotli compression middleware for API responses
- Events endpoint returns large JSON — compression reduces transfer significantly

### TypeScript Strict Mode Audit
- Eliminate all implicit `any` types
- Ensure strictNullChecks, noUncheckedIndexedAccess enabled
- Clean up all tsc warnings in the codebase

### Rate Limit Differentiation
- Per-endpoint rate limits reflecting actual usage patterns
- Flights: higher limit (aggressive polling), Events: lower (15-min cache), Geocode: very low
- Already partially done — needs consistency pass and documentation

### OpenAPI Spec
- Hand-written OpenAPI 3.0 spec for all /api/* endpoints
- Response schemas, error shapes, query params documented
- Serves as API documentation artifact in the repo

### Claude's Discretion
- Config validation strategy for serverless (crash vs warn for optional vars)
- Pino configuration details (log levels, transport, serializers)
- Wave ordering and parallelization of the 13 work items
- Which `it.todo()` stubs to remove vs convert to real tests
- OpenAPI spec file location and format

</decisions>
