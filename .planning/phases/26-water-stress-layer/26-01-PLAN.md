<task type="auto" tdd="true">
  <name>Task 1: Define WaterFacility types and stress utility module</name>
  <files>server/types.ts, src/lib/waterStress.ts, src/__tests__/waterStress.test.ts</files>
  <behavior>
    - stressToRGBA(0) returns [0, 0, 0, 200] (black = extreme stress)
    - stressToRGBA(1) returns [125, 211, 252, 200] (light blue = healthy)
    - stressToRGBA(0.5) returns midpoint color between dark blue and medium blue
    - stressToRGBA(-0.1) clamps to 0, stressToRGBA(1.5) clamps to 1
    - compositeHealth(5, 1.0) returns ~0 (extreme baseline stress, normal precip)
    - compositeHealth(0, 1.0) returns ~1 (no baseline stress, normal precip)
    - compositeHealth(2.5, 0.5) returns ~0.375 (moderate stress, dry conditions make it worse)
    - compositeHealth(2.5, 1.5) returns ~0.625 (moderate stress, wet conditions improve it)
    - bwsScoreToLabel(0-1) returns 'Low', (1-2) 'Low-Medium', (2-3) 'Medium-High', (3-4) 'High', (4-5) 'Extremely High'
  </behavior>
  <action>
    1. In `server/types.ts`, add these types (do NOT remove desalination from SiteType -- Plan 02 handles that):

    ```typescript
    export type WaterFacilityType = 'dam' | 'reservoir' | 'treatment_plant' | 'canal' | 'desalination';

    export interface WaterStressIndicators {
      bws_raw: number;    // baseline water stress raw value
      bws_score: number;  // 0-5 normalized score
      bws_label: string;  // human label
      drr_score: number;  // drought risk 0-5
      gtd_score: number;  // groundwater table decline 0-5
      sev_score: number;  // seasonal variability 0-5
      iav_score: number;  // interannual variability 0-5
      compositeHealth: number; // 0-1 (0=worst, 1=best)
    }

    export interface WaterFacility {
      id: string;           // "water-{osmId}"
      type: 'water';
      facilityType: WaterFacilityType;
      lat: number;
      lng: number;
      label: string;
      operator?: string;
      osmId: number;
      stress: WaterStressIndicators;
      precipitation?: {
        last30DaysMm: number;
        anomalyRatio: number;
        updatedAt: number;
      };
    }
    ```

    2. Create `src/lib/waterStress.ts`:
    - Export `STRESS_COLORS` array: 4 stops from [0,0,0] to [125,211,252]
    - Export `stressToRGBA(health: number, alpha?: number)`: linear interpolation across 4 color stops, clamp input to [0,1]
    - Export `compositeHealth(bwsScore: number, precipAnomalyRatio: number)`: baseline dominates (75%), precip adjusts (25%). Formula: `baselineHealth = 1 - (bws_score / 5)`, `precipModifier = clamp((precipRatio - 1.0) * 0.5, -0.25, 0.25)`, result = `clamp(baselineHealth + precipModifier, 0, 1)`
    - Export `bwsScoreToLabel(score: number)`: maps 0-5 score to WRI label strings
    - Export `WATER_STRESS_LEGEND_STOPS`: array of {color, label} for legend registration

    3. Create test file `src/__tests__/waterStress.test.ts` with tests for all exported functions using the behaviors listed above.
  </action>
  <verify>
    <automated>npx vitest run src/__tests__/waterStress.test.ts -x</automated>
  </verify>
  <done>WaterFacility types exported from server/types.ts. stressToRGBA, compositeHealth, bwsScoreToLabel all pass unit tests. STRESS_COLORS and WATER_STRESS_LEGEND_STOPS exported.</done>
</task>

    2. Create `scripts/extract-rivers.ts`:
    - Download Natural Earth 10m rivers+lake centerlines shapefile from `https://naciscdn.org/naturalearth/10m/physical/ne_10m_rivers_lake_centerlines.zip`
    - Extract the .shp/.dbf/.prj files from the ZIP
    - Use `shapefile` npm package to parse into GeoJSON features
    - Filter to 6 named rivers: Tigris, Euphrates, Nile, Jordan, Karun, Litani (match on `properties.name` or `properties.name_en`)
    - Apply Douglas-Peucker simplification at epsilon=0.05 degrees (same as Phase 25 ethnic zones) -- implement a simple DP function or use coordinates directly if the river data is already small
    - Write `src/data/rivers.json` as a GeoJSON FeatureCollection
    - Each feature should have `properties.name` (river name) and `properties.scalerank` if available
    - Target: under 200KB total file size
    - Log which rivers were found and the final file size

    3. Create `scripts/extract-aqueduct-basins.ts`:
    - Download WRI Aqueduct 4.0 data from `https://files.wri.org/aqueduct/aqueduct-4-0-water-risk-data.zip`
    - If the ZIP fails (URL may have changed), fall back to the WRI data page and log instructions
    - Extract the baseline annual CSV file from the ZIP
    - Filter to Middle East region basins (lat 0-50, lng 20-80) using available geography columns
    - For each basin, extract: `pfaf_id`, `bws_raw`, `bws_score`, `bws_cat`, `bws_label`, `drr_score`, `gtd_score`, `sev_score`, `iav_score`, `name_0` (country), and centroid coordinates if available
    - If basin polygon centroids are not in the CSV, include whatever geographic reference is available (country name + pfaf_id for nearest-centroid matching)
    - Write `src/data/aqueduct-basins.json` as an array of basin objects
    - Log basin count and file size

    4. Run both scripts: `npx tsx scripts/extract-rivers.ts && npx tsx scripts/extract-aqueduct-basins.ts`

    **IMPORTANT:** These scripts download from external URLs. If a download fails:
