    - Export `async function fetchPrecipitation(locations: {lat: number, lng: number}[]): Promise<PrecipitationData[]>`
    - Use Open-Meteo Forecast API: `https://api.open-meteo.com/v1/forecast?latitude=${lats}&longitude=${lngs}&daily=precipitation_sum&past_days=30&forecast_days=0&timezone=UTC`
    - Batch locations into groups of 100 to stay under API limits
    - For each location, sum the 30-day precipitation values
    - For anomaly ratio: use a rough regional monthly average (hardcode: ~20mm/month for arid Middle East, ~50mm/month for Fertile Crescent). This gives a reasonable relative signal without needing a climate API.

    3. Create `server/routes/water.ts`:
    - Export `waterRouter = Router()`
    - `GET /` — fetch water facilities with Redis cache:
      - Cache key: `water:facilities`
      - Logical TTL: 24h (WATER_CACHE_TTL)
      - Hard Redis TTL: 3 days (259200s)
      - On miss: call `fetchWaterFacilities()`, cache, return
      - On stale: return stale data, log
      - Follow exact pattern from `server/routes/sites.ts`
    - `GET /precip` — fetch precipitation for cached facilities:
      - Cache key: `water:precip`
      - Logical TTL: 6h (WATER_PRECIP_CACHE_TTL)
      - Hard Redis TTL: 1 day (86400s)
      - On miss: load water facilities from cache (or fetch), extract coordinates, call `fetchPrecipitation`, cache, return
      - Return as `CacheResponse<PrecipitationData[]>`
