## Decisions Made

- **FetchRecord per-store**: Defined the `FetchRecord` interface in each store file rather than a shared module. This avoids cross-store import dependencies and keeps each store self-contained per existing codebase conventions.
- **Optional message param**: `setError(message?: string)` with `'Unknown error'` default ensures all 18 existing callers work unchanged. New instrumented callers pass the actual error string.
- **Silent swallow preservation**: `useWaterPrecipPolling` intentionally does NOT call `setError()` on failure. Added `recordFetch(false, ...)` for observability without disturbing the `connectionStatus` or facility data. This matches the existing design intent documented in RESEARCH.md Pitfall 4.
- **Non-ok response as early return**: Changed from `throw new Error()` + catch to explicit `setError(msg)` + `recordFetch(false, ...)` + `return` for non-ok HTTP responses. This provides more precise error messages.
