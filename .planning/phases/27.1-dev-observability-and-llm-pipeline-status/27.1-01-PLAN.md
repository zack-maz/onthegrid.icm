**Part C: Create /api/events/llm-status endpoint (per D-02, D-08)**

Add a new GET route handler to the existing `eventsRouter` in events.ts. This endpoint is DEV-ONLY per the additional user decision, gated by `NODE_ENV !== 'production'`:

```typescript
eventsRouter.get('/llm-status', async (_req, res) => {
  // DEV-ONLY: gate this endpoint in production
  if (process.env.NODE_ENV === 'production') {
    return res.status(404).json({ error: 'Not found' });
  }

  // If in-memory progress is active (not idle), return it directly
  if (llmProgress.stage !== 'idle') {
    return res.json(llmProgress);
  }

  // Otherwise, fall back to Redis summary from last completed run
  try {
    const summary = await cacheGetSafe<LLMRunSummary>(LLM_SUMMARY_KEY, 0);
    if (summary?.data) {
      return res.json({ stage: 'idle' as const, lastRun: summary.data });
    }
  } catch {
    // Redis failure — return idle with no history
  }

  res.json({ stage: 'idle' as const, lastRun: null });
});
```

IMPORTANT: Place this route handler BEFORE the main `eventsRouter.get('/', ...)` handler so Express matches `/llm-status` before the catch-all `/` route.
