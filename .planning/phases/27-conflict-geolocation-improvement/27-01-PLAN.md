<objective>
Rewrite the ConflictEventType union from 11 CAMEO-derived types to 5 attack-vector categories, update all server-side consumers (CAMEO mapper, Goldstein ceilings, event scoring, config, cache response schema), and add LLM-related fields to ConflictEventData. This is the type foundation that every subsequent plan depends on.

Purpose: The 11-type CAMEO taxonomy is too granular and confusing for users. The 5-type system groups by attack vector (airstrike, on_ground, explosion, targeted, other) which maps to how conflicts are actually reported and understood.

Output: Updated server/types.ts, server/adapters/gdelt.ts, server/lib/eventScoring.ts, server/config.ts with new env vars, and updated server tests.
</objective>
