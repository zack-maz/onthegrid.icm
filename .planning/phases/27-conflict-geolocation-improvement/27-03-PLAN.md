**New constants:**

```typescript
const LLM_EVENTS_KEY = 'events:llm';
const LLM_PROCESS_KEY = 'events:llm-process-ts';
const LLM_COOLDOWN_MS = 900_000; // 15 minutes
const LLM_LOGICAL_TTL_MS = 900_000; // 15 minutes
const LLM_REDIS_TTL_SEC = 9000; // 2.5 hours hard TTL (same as raw GDELT)
```

**New helper functions (same pattern as shouldBackfill/recordBackfillTimestamp):**

```typescript
async function shouldRunLLM(): Promise<boolean> {
  try {
    const lastTs = await redis.get<number>(LLM_PROCESS_KEY);
    if (lastTs === null || lastTs === undefined) return true;
    return Date.now() - lastTs > LLM_COOLDOWN_MS;
  } catch {
    return true;
  }
}

async function recordLLMTimestamp(): Promise<void> {
  try {
    await redis.set(LLM_PROCESS_KEY, Date.now(), { ex: LLM_REDIS_TTL_SEC });
  } catch {
    /* best-effort */
  }
}
```

**Modified route handler logic:**

After the existing raw GDELT fetch + merge + prune produces `merged` array:

1. Check LLM cache first: `const llmCached = await cacheGetSafe<ConflictEventEntity[]>(LLM_EVENTS_KEY, LLM_LOGICAL_TTL_MS)`
2. If `llmCached && !llmCached.stale`, serve llmCached data and return early (before raw GDELT processing)
3. If `isLLMConfigured() && await shouldRunLLM()`:
   a. Group merged events: `const groups = groupGdeltRows(merged)`
   b. Diff against cached LLM events to find NEW groups (groups whose key doesn't match any cached event)
   c. Process new groups: `const enriched = await processEventGroups(newGroups)`
   d. If enriched is non-null, geocode: `const geocoded = await geocodeEnrichedEvents(enriched, newGroups)`
   e. Merge geocoded with cached LLM events
   f. Cache the merged LLM events: `await cacheSetSafe(LLM_EVENTS_KEY, llmMerged, LLM_REDIS_TTL_SEC)`
   g. Record timestamp: `await recordLLMTimestamp()`
   h. Serve LLM-enriched events
4. Else if llmCached exists (stale but present), serve stale LLM events
5. Else serve raw GDELT events (the existing merged array)

Wrap the entire LLM processing block in try/catch. On ANY failure, log.warn and fall through to serving raw GDELT. The map NEVER goes blank.

Keep the existing raw GDELT caching (events:gdelt key) unchanged. The LLM cache (events:llm) is a separate layer on top.
